# 实施计划：v0.9 局内结构：分支地图、撤退与保险

> 对应 PRD：[PRD/v0.9-run-structure.md](../PRD/v0.9-run-structure.md)
> 日期：2026-07-16。前置：v0.8（README 状态列**已实施**，代码核实一致）。状态：**待实施**。

## 代码核对结论（动笔前逐项核实）

**依赖确认**：v0.8 已实施——`gameStore.addSp` 在位、胜利浮层并列 `+SP` 且确认键为 `addIsk + addSp + addLoot + advanceDepth` 四连、`buildEncounter` 产出 `spReward`、`SAVE_VERSION` 仍为 **2**、balance.md 已有 SP 经济章节。不阻塞。

**PRD 引用逐项核实（与代码一致）**：

| PRD 引用 | 代码现状 | 结论 |
|----------|----------|------|
| App 三视图状态机 | `useState` 的 `view: 'station'\|'map'\|'space'`，另有 `nodeType`/`encounters` 两个局部态 | ✓ `runState` 取代后两者 |
| `goToMap` 预掷约定 | UNDOCK/胜利后重掷 `{patrol, elite}` 两个 encounter，"所见即所遇" | ✓ 预掷对象改为整段 |
| BattleScene outcome 分支 | `null \| 'victory' \| 'defeat'`，`endBattle` 冻结模拟、victory 时即刻 roll 掉落 | ✓ 撤退作第三值并列 |
| 玩家 HP 初始化 | `playerHp = useRef({...eff.defense 满值})`，电容同 render 初始化 | ✓ 初始 HP 入参挂点明确 |
| `shipDestroyed` | 摘除船体+fit、depth 重置 1、免费 Incursus 保底条款 | ✓ 赔付插在保底判定前 |
| `partialize`/`merge` | 只存 id；merge 显式列举 key（未列举的 persisted key 会被丢弃） | ✓ insurance 需同时加进两处 |
| MapScreen | `threatLabel`/`FactionIntel`/`primaryDamageTypes` 三个纯展示件 + 双圆盘布局 | ✓ 重写时保留前三者 |

**PRD 未明写、核对后必须落定的义务（本计划已按裁定编写；前两条需用户拍板）**：

1. **战斗中 header 的 ABORT & DOCK 是零代价逃生门**：`App.jsx` 对一切非 station 视图渲染该按钮，战斗里点击即回站——船、fit、深度全保留，比 FR-4 的 8 秒对齐撤退严格更优。**必须收口**（`view === 'space'` 时隐藏），否则撤退机制形同虚设。这是移除一个现存玩家可见行为，列入拍板项。
2. **深度推进从"每胜 +1"改为"段完成一次 +5"**：现状 `advanceDepth` 在每次胜利确认时 +1，与 FR-1"走完段末层回站，下一段从 depth+5 生成"叠加会双重推进。裁定：节点深度改由段图静态携带（`startDepth + 层号`），store 的 `deadspaceDepth` 只在段完成时一次性 +5（新 action `advanceSegment`，`advanceDepth` 移除）；撤退死路提前回站、ABORT、刷新均**不**推进深度——只有走完段末层才算。这是 PRD 隐含但未明写的行为变化（爬深节奏变成整段制），列入拍板项。
3. **PRD 写于 v0.8 之前**：结算现含 SP——撤退放弃的是**赏金 + SP + 掉落**三样；胜利仍逐节点即时入账（见设计决策 3 的时序表）。
4. **BattleScene 两处读 store 深度需改为节点深度**：`rollLoot(enemy, deadspaceDepth, nodeType)`（掉落品级门控）与直入 fallback 的 `rollEncounter(deadspaceDepth, ...)`。段内 store 深度恒为段起点，layer 4 的战斗若仍读 store 会把掉落品级压低 3 层——改为 `depth` prop（缺省回退 store 值，保住 fallback 语义）。
5. **insurance 存档形态简化**：PRD 草拟 `{ shipId, payout }`，裁定为 `{ shipId }`——payout 在理赔时按 `SHIPS[shipId].price` 现价计算，贴"存档只存 id、数值改动对旧档即时生效"的既有惯例（sellModule 按现价 50% 同理）。
6. **"卖船作废不退"无对应代码**：store 没有 sellShip action（市场只卖模块），该条款实际只落在 `switchShip`（换船作废）。
7. **runmap 输出形态微调**：PRD 草拟 `{ layers: [{ nodes: [...] }] }`，裁定扁平化为 `layers: node[][]`（层对象无其他字段，包装层是噪音）。

## 现状分析

- **结算通路**：`endBattle(result)` 冻结模拟 → outcome 浮层 → 确认键写 store + 导航回调。三种 outcome 共用此模式，撤退照搬（浮层确认制保持一致，读条完成 ≠ 立即切视图）。
- **BattleScene 每场重挂载**：App 以 `view === 'space' &&` 条件渲染，进出战斗即挂载/卸载——初始 HP 走 prop、结束 HP 走回调出参，天然成立，无需外部 store。段内 fit/技能不可变（FITTING/SKILLS 都在站内），`getEffectiveStats` 上限在段内是常量，HP 快照只存当前值合法。
- **电容**：`capRef` 每次挂载从 `eff` 满值初始化——FR-3"电容每场重置"即现状，零工作。
- **MapScreen 可整体重写**：双圆盘布局废弃；`threatLabel`/`FactionIntel` 两个展示件平移进新节点卡片。v0.8 验证记录的教训：**zustand 在 Node SSR 下恒读建店瞬间的初始状态**——新 MapScreen 改为**纯 props 组件**（runState、maxHp、回调全部下传，不再 `useGameStore`），B 组测试就能喂 fixture 直接断言，绕开该限制。
- **App 是本版重写面最大的文件**：但路线可行性判定（可达节点、死路、段完成）全部抽成 `runmap.js` 纯函数，App 只做"事件 → setRunState + 导航"的薄壳，重逻辑进 A 组测试。
- **保险与既有条款的耦合点**：`shipDestroyed` 的免费 Incursus 保底、`switchShip` 的剥装回库、`buyShip` 的 podded 自动登舰。逐一核过：赔付先入账再判保底（PRD 已点名）；switch 作废；podded 自动登舰时保单必已在死亡时消费——不变式"insurance 非空 ⇒ shipId === activeShip.id"在全部路径下成立，merge 再兜一层校验。

## 设计决策

### 1. `src/lib/runmap.js` — 段地图生成器（纯函数）

```js
export const SEGMENT_LAYERS = 5;

// rng 单流注入（沿用 rollEncounter 惯例）。roll 顺序即协议，锁定为：
// 逐层节点数 → 逐层逐节点类型 → 约束修正 → 逐层对连边 → 逐层逐节点内容
// （encounter / wreck 的掉落+伏击）。改动顺序 = 改动同种子输出，测试依赖此序。
export function generateRunMap(startDepth, rng = Math.random) => {
  startDepth,
  layers: [ /* SEGMENT_LAYERS 层 */ [ {
    id: 'l0n1',            // 'l{layer}n{index}'，全图唯一
    layer, index,
    type: 'patrol' | 'elite' | 'repair' | 'wreck',
    depth: startDepth + layer,          // 该节点的一切数值锚点
    encounter,             // 仅战斗节点：rollEncounter(depth, type, rng) 预掷
    loot,                  // 仅 wreck：预掷掉落 id[]（见决策 5）
    ambush,                // 仅 wreck：encounter | null（25% 预掷）
    edges: [0, 1]          // 指向下一层 layers[layer+1] 的下标；末层为 []
  } ] ]
}

// 运行期判定助手（App 薄壳的全部"重逻辑"）：
export function reachableNodeIds(map, atNodeId /* null=段起点 */, closedIds) => string[]
export function isFinalLayer(map, nodeId) => boolean
```

**每层节点数**：2 或 3，各 50%。**类型分布**（定稿）：

| 层 | patrol | elite | wreck | repair | 说明 |
|----|--------|-------|-------|--------|------|
| 1 | 100% | — | — | — | 满血起步的基准读数，修理/赌博节点在此无意义 |
| 2–4 | 55% | 15% | 20% | 10% | 风险/回报/续航的主体混合区 |
| 5（段末） | — | 100% | — | — | FR-2"段末固定精英"；2–3 个并列 elite，凭 intel 挑一个打 |

**约束修正（掷完后执行，A 组逐条断言）**：L2–4 若无 repair → 在 L3–4 的非 elite 节点中随机替换一个为 repair（消耗一次 rng；血线压力积累后、段末决战前的位置）；每层最多 1 个 repair（超出降级为 patrol，按节点序）;每段最多 2 个 repair（同上）。elite 不设上限——它就是风险旋钮。"每段 ≥1 精英"由段末层自动满足。

**连边（相邻层 A→B，A/B ∈ {2,3}，节点按下标序）**：
1. 基础覆盖：每个上层节点 i 连 `j*(i) = round(i × (B−1)/(A−1))`；
2. 补洞：无入边的下层节点 j 补一条来自 `round(j × (A−1)/(B−1))` 的边；
3. 加宽：每个上层节点对 j\*±1 两个方向各以 50% 掷一次（先 +1 后 −1），仅当与既有边集不交叉（∀ 既有 (i′,j′)：`(i′−i)(j′−j) ≥ 0`）时采纳。

不变式：无交叉边（平面图）、非末层出度 ≥1、非首层入度 ≥1、全图自第 1 层可达（归纳自入度约束）。

### 2. run 状态机（`App.jsx` 重写）

`runState`（`useState`，**不入 store 不入档**）：`null`（不在段中）或

```js
{ map,                       // generateRunMap 输出
  at: null | nodeId,         // 当前位置；null = 段起点（第 1 层全部可选）
  playerHp: null | {shield, armor, hull},   // null = 满血；快照存当前值，上限归 getEffectiveStats
  cleared: [nodeId],         // 已清除（胜利/修理/拾取）——渲染用
  closed: [nodeId],          // 撤退封闭——不可再入
  notice: null | { nodeId, lootIds } }      // 残骸场拾取结果浮层（瞬态）
```

流转表（事件 → runState / store / 视图）：

| 事件 | runState | store | 视图 |
|------|----------|-------|------|
| UNDOCK | 生成 map；at=null, hp=null | — | map |
| 点战斗节点（patrol/elite/伏击 wreck） | 记录目标节点 | — | space |
| 胜利确认（非段末） | at=目标；cleared+=；hp=回传快照 | `addIsk+addSp+addLoot`（现状，即时入账） | map |
| 胜利确认（段末） | 置 null | 同上 + `advanceSegment()`（+5） | station |
| 撤退确认（`reachableNodeIds` 非空） | closed+=目标；hp=快照；at 不变 | **零写入** | map |
| 撤退确认（为空 = 死路） | 置 null | 零写入（深度不推进） | station |
| 阵亡确认 | 置 null | `shipDestroyed()`（含保险赔付，深度重置 1） | station |
| 点 repair 节点 | at=目标；cleared+=；**hp=null** | — | map（就地生效） |
| 点 wreck（ambush=null） | at=目标；cleared+=；notice={loot} | `addLoot(node.loot)` | map（浮层展示） |
| DOCK（map 视图按钮/header，`window.confirm`） | 置 null | 零写入 | station |
| 刷新/关页 | 内存态丢失 | 已入账收益在档（每次 set 即写档） | station（重开） |

- header 的 ABORT & DOCK：`view === 'space'` 时**隐藏**（核对结论 1）；map 视图保留，语义 = 弃段（confirm 提示"已结算收益保留、地图作废、深度不推进"）。
- 段末胜利浮层按钮文案由 `isFinalNode` prop 切换为 "Loot Wreck & Dock"（App 知道层位，BattleScene 不必懂地图）。

### 3. HP 跨节点持续与结算时序（v0.8 交界，PRD 特别标注项）

- `src/lib/combat.js` 新增纯函数（BattleScene 用它初始化，A 组直测）：

  ```js
  // 段内 HP 快照恢复：null = 满血开场；快照逐层夹紧到当前有效上限
  //（防御性——段内上限本为常量，夹紧只防手工构造的异常入参）。
  export function resumeHp(snapshot, defense) {
    if (!snapshot) return { shield: defense.shield.hp, armor: defense.armor.hp, hull: defense.hull.hp };
    return {
      shield: Math.min(snapshot.shield, defense.shield.hp),
      armor: Math.min(snapshot.armor, defense.armor.hp),
      hull: Math.min(snapshot.hull, defense.hull.hp)
    };
  }
  ```

- BattleScene：`playerHp = useRef(resumeHp(initialHp, eff.defense))`，HUD 初值同源；电容不入快照（每场满，现状即是）。
- **结算时序定稿**：ISK/SP/掉落**逐节点即时入账**（现状不变，胜利确认键写 store，persist 即写档）；HP 快照**只回传 App 的 runState**（不入 store 不入档）；深度**只在段完成时 +5**。三种资源三种时序，刷新语义因此自洽：刷新丢段图与 HP、保收益、深度停在段起点——重开 UNDOCK 从同深度重掷新段，无利可图也无损失放大。
- 回传方式：`onVictory({ ...playerHp.current })` / `onRetreat({ ...playerHp.current })`（浅拷贝防 ref 逃逸）；`onDefeat()` 无参。

### 4. FR-4 撤退（ALIGN & WARP）

- **交互**：TACTICAL COMMANDS 行右端新增琥珀色按钮 `ALIGN & WARP`；点击开始 8.0 秒读条（`ALIGN_TIME = 8`，文件顶常量），再点取消（EVE 停船断对齐的语义）；读条期间移动指令、模块、敌方 AI 全部照常——**唯一新增模拟逻辑是一个计时器**。
- **实现（refs 惯例，零 React 态入模拟）**：`alignRef = useRef(null)`（null | 剩余秒）；`stepRef` 内 `alignRef.current -= dt`，归零 → `endBattle('retreat')`；HUD 经既有 `syncHud`（0.15s 节流）同步剩余秒到一个 `alignHud` state，战斗信息条显示 `WARPING IN {ceil}s`（琥珀色）；开始/取消/完成各写一条 combatLog。`battleOverRef` 冻结后按钮失效（endBattle 已置位，天然免疫胜利/阵亡同帧竞态：hull 归零分支先于对齐判定执行，同帧同时满足时阵亡优先）。
- **第三 outcome**：`endBattle('retreat')` 不 roll 掉落；浮层（琥珀边）文案 "WARP OUT — Engagement abandoned. Bounty, SP and salvage forfeited; the site is lost for this dive."，确认键 `onRetreat(hp快照)`。
- **无安全阀（PRD 开放问题 2 落定）**：不做武器离线、不做电容门槛。理由：代价已有四重——8 秒白挨打、放弃该节点三样收益、节点封段内不可再入、可能触发死路提前回站（深度不推进）；网状克制（web 局里 8 秒 ≈ 死刑）正是设计想要的"撤退不是万能键"。若实测出现"远程局无脑撤"，balance.md 已记录两个备选阀门，先调不改架构。
- **模块 type 分支零新增**：撤退是指令/计时器，不是模块——v0.7 立的管线合规审计继续通过。

### 5. FR-2 非战斗节点（地图层逻辑，不进 BattleScene）

- **维修锚点（repair）**：免费修满（PRD 开放问题 4 落定：倾向已明说，节点占位本身就是代价——一个 repair = 一场没打的仗；ISK 计价修理等 v0.11 弹药经济一起评估）。实现即流转表一行：`playerHp = null`。卡片文案 "REPAIR ANCHOR — Hull & armor restored"（青色）。
- **残骸场（wreck）**：生成时预掷（rng 单流，全节点确定性）：
  - 掉落：`pickWeightedNpc(depth, rng)` 取一个当层加权 NPC，`rollLoot({ lootTable: npc.lootTable }, depth, 'patrol', rng)` 单轮——零新数据表，品级门控天然按**节点深度**走，"谁的残骸"随深度加权池变化；空掉落合法（浮层显示 "Nothing of value"）。
  - 伏击：25% 预掷 `ambush = rollEncounter(depth, 'patrol', rng)`，否则 null。**伏击替代拾取**（残骸是诱饵的叙事；不叠发免双份收益）：入节点直接进战斗，`introLog` prop 显示 "Ambush! Hostiles decloak among the wrecks."（BattleScene 新增可选 prop，缺省现文案），胜利按正常巡逻结算（赏金+SP+掉落）。
  - 卡片文案 "DEBRIS FIELD — Salvage · sensor ghosts"（琥珀色），**不**泄露 ambush 已预掷的真值——赌的就是这个。
- 两类节点均不产 SP/赏金（repair/wreck 拾取无战斗），与 balance.md SP 章节"SP 表达战斗经验"自洽。

### 6. FR-5 船体保险（`gameStore`）

- **形态**：`initialState` 增 `insurance: null`；有单时 `{ shipId }`（核对结论 5：payout 理赔时按现价算）。不变式：非空 ⇒ `shipId === activeShip.id`。
- **数值（定稿，入 balance.md）**：保费 = `round(price × 0.3)`，赔付 = `price × 1.0`（白金档）；单船单张，不可叠加。
- **actions**：

  ```js
  buyInsurance: () => set(state => {
    if (!state.activeShip || state.insurance) return state;
    const premium = Math.round(SHIPS[state.activeShip.id].price * 0.3);
    if (state.isk < premium) return state;
    return { isk: state.isk - premium, insurance: { shipId: state.activeShip.id } };
  }),
  ```

  `switchShip`：成功换船的返回对象加 `insurance: null`（跟船作废，不退；同 id 重复登舰已被现有守卫挡住，不构成洗单路径）。`shipDestroyed`：入口先算 `payout`（有单且 shipId 匹配 → `SHIPS[id].price`，否则 0），`isk + payout` 参与后续免费 Incursus 保底判定（赔付先入账，PRD 点名的顺序），两条返回路径都带 `insurance: null` 与结算后的 `isk`。`advanceSegment: () => set(s => ({ deadspaceDepth: s.deadspaceDepth + SEGMENT_LAYERS }))`（`advanceDepth` 移除；`SEGMENT_LAYERS` 自 runmap 导入，无环：runmap → data/lib，不触 store）。
- **UI（FittingWindow 中央 `center-ship-info` 下方，3–8 行）**：未投保 → 按钮 `Insure hull — {premium} ISK (pays {price} on loss)`（余额不足禁用）；已投保 → 绿色文本 `✓ Insured — pays {price} ISK on loss`；podded 视图不渲染。选 FittingWindow 而非市场：保单跟"正在开的这艘"，出击前的装配界面是决策现场。
- **净损失锚点自检**：投保 Incursus 全损 = 18,000（保费）+ fit（T1 满配约 40–60k）——仍 ≈ 2–4 场巡逻赏金，损失锐度保住；未投保与现状逐字节一致。

### 7. MapScreen 重写（纯 props，一屏 5 列）

- **Props**：`{ runState, maxHp, onSelectNode(nodeId), onDismissNotice, onDock }`——零 `useGameStore`（现状分析里的 zustand SSR 教训），`maxHp` 由 App 从 `getEffectiveStats` 算好下传。
- **布局**：水平 5 列（层序左→右），列头 `LAYER {k} · DEPTH {depth}`；每列 2–3 张节点卡（约 190×130px，5 列 ×3 行含边距 ≈ 1150×520px，1280 宽一屏放下，满足"不滚动"非目标）；卡片下层一张绝对定位 SVG 画连边。
- **节点卡**：战斗节点 = 类型徽标（PATROL 蓝 / ELITE 红虚线，沿用现配色基因）+ NPC 名 + `FactionIntel`（派系+伤害徽标，平移复用）+ `threatLabel`（射程/Web）+ 赏金——intel 完整度与现状逐项持平（验收 1）；repair/wreck 卡见决策 5 文案。

- Segment: 5 layers (SEGMENT_LAYERS, src/lib/runmap.js), 2–3 nodes per layer

  the only way depth advances. Retreat dead-ends, ABORT & DOCK and page refresh
  keep depth at d; death still resets to 1.
- Node type weights — layer 1: patrol 100%; layers 2–4: patrol 55% / elite 15% /
  wreck 20% / repair 10%; final layer: elite 100% (2–3 side by side, you fight
  the one you route into).
- Distribution constraints (post-roll fixes): ≥1 repair in layers 2–4, forced
  into a non-elite slot in layers 3–4 if none rolled; ≤1 repair per layer;
  ≤2 per segment. Elites are uncapped — they are the risk dial.
- Repair anchor: free full restore (all three layers). Its cost is the node slot
  itself — a repair taken is a fight's rewards skipped. ISK-priced repair is
  deliberately deferred to the v0.11 ammo-economy pass.
- Wreck field: one pre-rolled rollLoot pass against a depth-weighted NPC's
  lootTable at the node's own depth (patrol single pass, tier gates apply).
  Ambush chance 25%, pre-rolled at generation: an ambush replaces the salvage
  with a normal patrol-grade fight at node depth (full bounty/SP/loot on win —
  the wreck was bait).
- Retreat (ALIGN & WARP): 8.0s align (ALIGN_TIME, BattleScene), fully vulnerable,
  modules stay online, no capacitor gate, cancellable. Cost = the node's bounty +
  SP + salvage forfeited, node closed for the segment, possible early segment end
  (no depth advance). No extra penalty valve in v0.9 — webs making the 8s lethal
  up close is the intended counterplay. If "always retreat" dominates ranged
  fights in play, candidate valves (weapons offline while aligning, or a large
  capacitor activation cost) go in before any number changes.
- Insurance (platinum only): premium = 30% of hull price, payout = 100% of hull
  price on destruction, fittings never covered. One policy at a time, bound to
  the boarded hull; switching hulls voids it without refund; consumed on payout.
  Net insured loss = 30% hull + full fit ≈ 2–4 patrol bounties — smoothed, still
  sharp. Payout lands before the free-Incursus soft-lock check.
- HP persists across nodes within a segment; capacitor refills every fight. Run
  state (map/position/HP) is deliberately not persisted: a refresh is a voluntary
  dock-out — banked ISK/SP/loot keep (they settle per node), the segment is lost.
