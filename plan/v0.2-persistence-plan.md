# 实施计划：v0.2 存档与持久化

> 对应 PRD：[PRD/v0.2-persistence.md](../PRD/v0.2-persistence.md)
> 日期：2026-07-14

## 现状分析

- 全部玩家状态集中在 `src/store/gameStore.js` 单一 Zustand store（zustand v5，persist middleware 可用）。
- `inventory` 存**完整模块对象**（且同款重复条目是同一对象引用）；`activeShip` 是 `{ ...SHIPS.incursus, fittedModules: { high/mid/low: [模块对象] } }`。
- 组件（FittingWindow / StationHub / BattleScene）全部经 store selector 读取对象 —— 只要运行时 store 结构不变，持久化改造可以**零组件改动**（Reset 按钮除外）。
- `blueprints` 目前是静态内容（`manufacture` 不修改它），不应入档，否则会冻结旧定义。

## 设计决策

### 1. persist 配置（FR-1）

`persist(config, options)` 包裹 store，localStorage key = `eve-rogue-save`，`version: 1`。

### 2. 存档格式：只存 id（FR-2）

`partialize` 把状态转成纯 id 形式，落盘示例：

```json
{
  "state": {
    "isk": 50000,
    "sp": 1500,
    "skills": { "engineering": 1, "gunnery": 2, "navigation": 1 },
    "deadspaceDepth": 3,
    "inventory": ["rocket_launcher_i", "stasis_webifier_i"],
    "activeShip": {
      "shipId": "incursus",
      "fitted": { "high": ["light_electron_blaster_i", "light_electron_blaster_i"], "mid": [], "low": [] }
    }
  },
  "version": 1
}
```

- 同款多件 = id 重复出现，天然支持。
- 模块/舰船数值改动后，旧档读到的是新定义（PRD FR-2 的核心诉求）。
- `blueprints` 不入 partialize，每次启动取初始值。

### 3. 反序列化与容错（FR-2 / FR-4）

自定义 `merge(persisted, current)`：

- id → 对象：查 `MODULES` / `SHIPS`，未知 id `filter(Boolean)` 静默丢弃（FR-4）；未知 shipId 回退 Incursus。
- `skills` 与默认值浅合并（未来新增技能自动补默认等级）。
- 数值字段用 `Number.isFinite` 校验，非法则取默认。
- 整体 try/catch，任何异常 → console.warn + 返回初始状态（不白屏）。

### 4. 版本迁移与损坏兜底（FR-3）

- `migrate: () => undefined`：版本不匹配时 zustand 才调用 migrate，返回 `undefined` 即丢档，`merge` 收到空值走默认分支 → 新开局。v0.3 需要真迁移时再写分支。
- 不用 `createJSONStorage`，改传自定义 `storage` 对象，`getItem` 内 `JSON.parse` 包 try/catch：localStorage 里是乱码时返回 `null`（等同无存档），显式覆盖 FR-3 的"损坏优雅降级"。

### 5. 重置入口（FR-5）

- store：提取 `initialState()` 工厂函数（`create` 初始化与 reset 共用），新增 `resetProgress` action = `set(initialState())`，persist 自动落盘覆盖。
- StationHub 导航栏右侧加 "RESET PROGRESS" 按钮（`marginLeft: 'auto'`、红色调），`window.confirm` 二次确认后调用（与现有 `alert` 风格一致，本阶段不引入弹窗组件）。

## 实施步骤

1. **`src/store/gameStore.js`**：提取 `initialState()`；`persist` 包裹（name/version/storage/partialize/migrate/merge，含 `toIds`/`fromIds` 辅助函数）；新增 `resetProgress`。
2. **`src/components/StationHub.jsx`**：导航栏加 Reset 按钮 + confirm。
3. **`.claude/launch.json`**：补 dev server 启动配置（端口 5173），用于浏览器验证。

## 验证计划（对照 PRD 验收标准）

在浏览器中依次执行：

1. **正常持久化**：训练技能（SP 1500→500）、制造模块（ISK 变化）、装配 2 件模块 → 刷新 → 全部保留。
2. **损坏兜底**：`localStorage.setItem('eve-rogue-save', '{oops')` → 刷新 → 正常新开局、控制台无未捕获异常。
3. **版本丢档**：手写 `version: 99` 的存档 → 刷新 → 新开局。
4. **无效 id 清理**：存档 inventory 中塞入 `"deleted_module"` → 刷新 → 该条目消失，其余正常。
5. **重置**：点击 Reset（confirm 通过）→ 回到 50,000 ISK / 深度 1 / 初始库存。
6. `npm run lint` 通过。

## 风险

- Vite HMR 会重建 store 并触发 rehydrate —— 行为正确（读档），反而改善开发体验，无需处理。
- `window.confirm` 与现有 `alert` 同属临时 UI，v0.3 市场版可统一换成游戏内弹窗（不在本版范围）。
