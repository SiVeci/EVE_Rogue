# PRD v0.9 — 局内结构：分支地图、撤退与保险

## 概述

把"一局"从无限线性爬深改造成有形状的下潜段：出击时生成 Slay the Spire 式分支地图（一段 5 层），玩家 HP 跨节点持续，节点类型扩展（维修锚点、残骸场），战斗中可对齐脱离（撤退），站内可为船体投保。主题：让**路线选择与风险管理**成为装配之外的第二决策层。

## 背景与动机

- 当前地图每层只有并列 2 节点（patrol / elite），且每场战斗满血开场——选精英只是纯粹的奖励赌博，没有资源代价；深度无限累进，"一局"没有节奏单元，roguelike 的成分实际很薄。
- 战斗打不过只能死：没有中途脱离手段。v0.7 之后敌情差异已经很大（射程、速度、EWAR、伤害类型），"这个敌人克制我这套 fit"的局面会正常出现，唯一出路是白送船，与"看敌情决策"的主线矛盾。
- 阵亡损失 = 船体 + 全部已装模块，损失尖锐且随 T2 掉落越来越重；保险是 EVE 原生机制，可以平滑船体部分的损失曲线，而不动装配损失的痛感。

## 目标

- 出击前能看到整段路线图，"这条路两个精英但通向维修锚点"式的权衡真实存在。
- HP 成为段内的持续资源：修复手段（维修节点、主动修盾/修甲模块、提前回站）各有代价，主动修复模块的装配价值随之上升。
- 打不过可以跑：撤退有明确代价（放弃该节点奖励、节点封闭）但保住船和装配。
- 阵亡的 ISK 曲线被保险平滑：保费成为常态支出，全损不再一夜清零，但净损失依然显著。

## 功能需求

### 分支地图（下潜段）

- **FR-1 段地图生成**：UNDOCK 生成一段 **5 层**的分支图：每层 2–3 节点、层间连边（StS 式，只能沿边走到下一层）；走完段末层回站结算，下一段从 `depth + 5` 继续生成。阵亡仍重置 depth 1。地图 UI 替换现有双圆盘 MapScreen；节点上保留完整敌情 intel（派系、伤害类型徽标、武器射程、赏金）——"所见即所遇"约定不变，战斗节点在生成时即预掷 encounter。
- **FR-2 节点类型**：`patrol`（现状）、`elite`（现状）、**维修锚点**（非战斗：免费修满或低价按比例修，二选一实现）、**残骸场**（非战斗：直接 roll 一次当层掉落，小概率触发伏击转为战斗，伏击敌情用当层正常 `rollEncounter`）。分布规则：每段至少 1 个维修锚点、至少 1 个精英；**段末层固定为精英**。
- **FR-3 HP 跨节点持续**：玩家盾/甲/结构在段内跨战斗保留（电容每场重置满，避免节奏拖沓）；run 状态由 App 层持有。本版**不持久化 run 中间态**——刷新/关页视为回站，已结算的 ISK/SP/掉落保留（现状即逐节点即时入账，不变）。

### 撤退与保险

- **FR-4 战斗中撤退**：新指令 **ALIGN & WARP**：8 秒对齐读条，期间正常受击、模块正常工作；读条完成脱离回地图。该节点标记"未清除"、本段不可重入，只能走其他边；若该层无其他可达节点则段提前结束回站。放弃该节点赏金与掉落。
- **FR-5 船体保险**：站内为当前船体购买一次性保单：保费 = 船价 30%，阵亡自动赔付船价 100%（EVE 白金档），装配模块不赔；保单跟船，换船/卖船作废不退。锚点入 balance.md：投保后全损净损失 = 30% 船价 + 全部 fit——仍显著痛，但不再是"从头再来"。

## 非目标

- run 中间态存档持久化（先验证结构好不好玩，再决定是否值得存档版本升级与状态快照的复杂度）。
- Boss 节点 / 段末守门人原型（候选池——段末先用 elite 顶位，原型化留给后续内容版本）。
- 文本选项式随机事件节点、多敌人遭遇。
- 超过一屏的大地图（5 层 × 最多 3 节点封顶，一屏放下、无滚动）。
- 撤退的更高级形态（战术转移到同层其他节点、代价性跳跃燃料等）。

## 技术方案要点

- 地图生成为纯函数 `src/lib/runmap.js`：`generateRunMap(depth, rng?)` 输出 `{ layers: [{ nodes: [{ id, type, encounter?, edges }] }] }`；rng 注入沿用 `rollEncounter` 的测试惯例。战斗节点的 encounter 在生成时预掷（保持 App `goToMap` 现有约定，只是掷的对象从 2 个变成整段）。
- `App.jsx` 三视图状态机扩为持有 `runState`（地图、当前节点、playerHp 快照）；BattleScene 接受**初始 HP 入参**并在结束时回传剩余 HP（胜利/撤退两条路），最大 HP 仍由 `getEffectiveStats` 唯一权威——快照存的是当前值，不是上限。
- 撤退在 BattleScene 内是与 victory/defeat 并列的第三种 outcome：warp-out 读条状态走 refs 惯例（不进 React state），不引入新模块类型分支。
- 保险入 `gameStore`：`insurance: { shipId, payout } | null` + 买保 action；`shipDestroyed` 内结算赔付。存档 `partialize` 增一个 key、`merge` 缺省 null——**零版本升级**。与"免费 Incursus 保底"条款并存：赔付先入账，保底条件（无船且 ISK 不足最便宜船价）随后判定，逻辑天然兼容，测试覆盖两者同时命中的顺序。
- 维修锚点/残骸场是地图层逻辑（改 HP 快照 / 调 `rollLoot`），不进 BattleScene。

## 验收标准

- [ ] 出击可见 5 层分支图与节点敌情，只能沿边走；段末（固定精英）打完回站，再出击时图从 depth+5 生成。
- [ ] 打完一场剩 30% 装甲 → 下一场以 30% 装甲开场；经维修锚点后修满；回站再出击满血。
- [ ] 每段至少 1 维修 + 1 精英的分布约束成立（生成器 Node 测试，多种子断言）。
- [ ] 战斗中启动撤退：8 秒读条期间仍受击；完成后回地图，该节点封闭、赏金未入账、船与装配完好。
- [ ] 投保后阵亡：ISK 增加额 = 船价，保单消失，深度重置与装配损失行为不变；未投保阵亡与现状完全一致。
- [ ] 残骸场入账一次掉落 roll；伏击触发时进入战斗且敌情与当层深度一致。
- [ ] 刷新页面：段丢弃、回到站内、已结算收益保留、存档无损坏（v2 存档往返测试）。

## 风险与开放问题

- **平衡基准漂移**：现有数值按"每场满血"调校，HP 持续会显著抬高主动修复模块与维修节点的价值（这正是目的），但可能把无修装配推出可行区——先观察，必要时给非战斗节点加少量被动回复而不是回调敌人数值。
- **撤退定价**：8 秒对齐在被 web 的近战局里约等于死刑（合理的克制关系），在远程局里几乎无代价——若实测成为万能保命键，备选安全阀：对齐期间武器离线，或消耗大量电容作为启动门槛。
- **段长 5 层是拍的**：3 层没有资源弧线，7+ 层单段过长（一段 10–20 分钟为目标带），先 5 后调。
- 维修锚点选"免费修满"还是"按 ISK 修"：前者实现最简、后者把 ISK 汇接进 run 内。倾向先做免费修满（节点稀缺性本身就是代价），弹药系统（v0.11）落地后再评估 ISK 修理。
