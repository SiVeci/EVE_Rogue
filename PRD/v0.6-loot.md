# PRD v0.6 — 战利品掉落

## 概述

胜利后从敌人残骸掉落模块：按 NPC 掉落表 roll，深度解锁更高品级，Meta 及以上品级**只出自掉落**（市场只卖 T1）。这是 roguelike"这局捡到了什么"核心动力的落点，也是后续内容大扩充的投放渠道。

## 背景与动机

- 当前战斗奖励只有 ISK，装备成长完全走市场购买，"越深越值得打"缺少数值以外的体现。
- v0.3 市场 + v0.5 被动模块就位后，模块池已有足够广度支撑掉落表分层。
- 经济定位必须清晰：市场 = 保底重建（治软锁），掉落 = 超额成长（追求动力）。两渠道不能互相蚕食。

## 目标

- 每场胜利有"开箱"时刻；elite 节点与更深深度有明确的掉落期待差。
- Meta/T2 品级形成"只能打出来"的稀缺性。
- 阵亡损失已装配模块（含打出来的珍品）的痛感，与"再下去捞"的动力互为张力。

## 功能需求

- **FR-1 掉落表**：`npcs.js` 每个 NPC 增加 `lootTable`：`[{ moduleId, chance }]`。胜利时逐条独立 roll。建议基线：普通节点合计期望 ~0.4 件/场，elite 翻倍（roll 两次同表）。
- **FR-2 品级分层**：掉落池按深度解锁：深度 1–3 掉 T1；4–6 加入 Meta（v0.3/v0.5 已有的 meta 变体归入此层）；7+ 加入 T2（本版先放 2–3 件 T2 样例：如 Light Neutron Blaster II、Ballistic Control System II，完整 T2 线在 v0.7 铺开）。
- **FR-3 战利品结算界面**：胜利后 BattleScene 显示结算浮层：ISK 入账 + 掉落清单（品级用颜色区分），确认后模块入机库、返回地图。没有掉落也显示 ISK 结算（保持节奏一致）。
- **FR-4 市场规则收紧**：市场货架固定为 T1；Meta/T2 不可购买，但**可以出售**（掉落变现的出口，建议售价为同类 T1 的 2–4 倍）。
- **FR-5 掉落即所得**：不做拾取判定/舱容限制，胜利即入库（残骸打捞玩法留给未来）。

## 非目标

- 弹药/矿物等原材料掉落与打捞（salvage）系统。
- 蓝图掉落驱动的工业深化（候选放 v0.7 或更后）。
- 装备词缀/随机属性（Abyssal roll），远期候选。

## 技术方案要点

- 掉落 roll 放在胜利结算处（当前 BattleScene 胜利分支），结果写入 store 新 action `addLoot(moduleIds)`；roll 逻辑抽成 `src/lib/loot.js` 纯函数（输入 npc + depth + nodeType，输出 moduleId 列表），便于调参和未来单测。
- 品级字段：`modules.js` 已有 `tier: 'T1'`，Meta/T2 沿用该字段，结算 UI 与市场过滤都读它。
- 受影响文件：`src/data/npcs.js`、`src/data/modules.js`（T2 样例）、`src/components/BattleScene.jsx`（结算浮层）、`src/components/MarketWindow.jsx`（T1 过滤 + Meta/T2 售价）、`src/store/gameStore.js`、新建 `src/lib/loot.js`。

## 验收标准

- [ ] 连打 10 场普通节点，掉落数量符合期望量级（~4 件）；elite 明显更多。
- [ ] 深度 2 打不出 Meta；深度 5 能打出 Meta；深度 8 能打出 T2（可临时调高概率验证）。
- [ ] 结算浮层正确显示 ISK 与掉落，确认后机库出现对应模块。
- [ ] 市场买不到 Meta/T2，但能以更高价卖出。
- [ ] 掉落的模块装配后阵亡 → 正常销毁（与 v0.3 死亡规则一致）。

## 风险与开放问题

- 掉落率与市场定价的耦合：若 T1 掉太多，市场购买失去意义 → 掉落表向 Meta 倾斜、T1 掉落只给低价值件。
- elite "roll 两次"还是"独立更优表"：先用 roll 两次（实现最简），体感不足再换。
