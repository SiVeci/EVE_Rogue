# PRD v0.8 — SP 经济闭环与 v0.7 债务清偿

## 概述

给技能点（SP）一个持续获取来源（战斗胜利产出），把 v0.7 建成的技能门槛体系（T2 装备、驱逐舰驾驶）从"不可达的静态内容"变成可规划的长线追求；同时清偿 v0.7 记录的两笔内容债：4 个 v0.4 老 NPC 的派系抗性签名补授、第二艘驱逐舰 Cormorant。小版本，纯数据 + 少量结算逻辑。

## 背景与动机

- **SP 目前没有任何获取来源**：初始 1500 点（`gameStore` 的 `initialState`），唯一操作是 `trainSkill` 消耗。训练价为 `1000 × (当前等级 + 1)`（`StationHub.trainCost`，锚点见 balance.md）：门槛技能第 1 级 1,000 SP，属性技能升一级起步 2,000 SP——初始 SP 只够点一次门槛技能，然后永远停摆。T2 武器门槛（武器技能 0→5）累计 15,000 SP、T2 支援件门槛（0→4）累计 10,000 SP，v0.7 铺的整条 T2 掉落线在实际游玩中永远解锁不了。这与 v0.3 修复的"ISK 经济软锁"同构，是当前最优先的结构性缺口（v0.7 计划原文即注明"胜利加 SP 的来源不在本版范围"）。
- v0.7 明确记录了两笔刻意推迟的内容债：**老 NPC 抗性签名**（Guristas ×2、Serpentis Brawler、Angel Webber 仍是 v0.4 的通用分层抗性，没有派系弱抗身份）和 **Cormorant**（观察 Catalyst 平衡后再做第二驱逐）。前者是 v0.11 弹药系统的硬前置——"按敌情换伤害类型"要求所有派系都有明确弱抗；后者的前提（Catalyst 管线与驱逐舰锚点已落地）现已成立。

## 目标

- 每场胜利同时产出 ISK 与 SP，技能训练成为与装备购买并行的常态成长轴。
- T2 门槛（15,000 SP 量级）成为可规划的中长线目标：正常游玩约 30–50 场胜利达成第一个 T2 解锁；Destroyers I（1,000 SP）约 5 场内可达，作为早期小目标。
- 4 个派系全部拥有成对的"输出伤害 / 自身弱抗"签名，MapScreen 敌情 → 换抗决策对每个敌人都成立。
- 驱逐舰级出现第二种 build 方向（Cormorant 远程磁轨 vs Catalyst 近战爆能）。

## 功能需求

- **FR-1 SP 战斗产出**：`buildEncounter` 增加 `spReward`，与 `reward` 同构缩放——建议基线 `200 × depthMult`，elite ×2（锚点写入 balance.md）。胜利结算浮层与 ISK 并列显示 `+SP`，确认时经 store 新 action `addSp(amount)` 入账。阵亡不给 SP（见开放问题）。
- **FR-2 SKILLS 页签目标导向**：门槛技能行显示它解锁什么（从 `SHIPS`/`MODULES` 的 `requiredSkills` 反查，如 "Small Hybrid Turret 5 → 125mm Railgun II, Light Neutron Blaster II…"），让 SP 花销有明确指向；属性技能行维持现状。
- **FR-3 老 NPC 抗性签名**：按 balance.md 已立的签名模式（弱抗类型比其余三系低 20–30 点、贯穿三层）retrofit 4 个 v0.4 NPC：Guristas Pirate / Guristas Sniper → 弱 KIN；Serpentis Brawler → 弱 TH（对齐 Scout）；Angel Webber → 弱 EXP（对齐 Hunter）。同时把老 NPC 盾面 `em: 0` 的 EVE 通用洞收窄，避免"打盾恒选 EM"盖过签名设计（v0.7 四个新 NPC 已采用的扁平化盾抗做法）。
- **FR-4 Cormorant**：Caldari 驱逐，7H/4M/2L（槽位总数 13，符合驱逐锚点），远程磁轨炮台定位、盾倾向——与 Catalyst（8H 近战爆能、甲/结构缓冲）形成射程轴上的镜像。`requiredSkills: { destroyers: 1 }`，数值按 balance.md 驱逐舰锚点填表（EHP 护卫带宽、agility 4.0–4.5s、sig 60–70m、价格 2.5–3× 护卫，落 ~190,000 ISK 档）。
- **FR-5 balance.md 增补 SP 经济章节**：SP 产出公式、各门槛的累计 SP 对照表（Destroyers I = 1,000 / T2 支援 = 10,000 / T2 武器 = 15,000）、"多少场胜利到哪个门槛"的节奏目标带。

## 非目标

- SP 的其他来源（事件节点、任务——留给 v0.9 的节点类型扩展做投放口）。
- 数值型技能扩展（武器技能附带 +伤害% 等）——v0.7 已决定门槛技能不给数值加成，先观察 SP 产出节奏再议，避免与 `skillMult` 管线二次纠缠。
- 老 NPC 的武器 / AI / 赏金改动（只动抗性签名，不动输出端）。
- 训练价公式调整（`1000 × (等级 + 1)` 维持，产出端调节奏）。

## 技术方案要点

- SP 产出走现有结算路径：`buildEncounter`（`src/data/npcs.js`）加 `spReward` 字段，BattleScene 胜利浮层确认处 `addIsk` 旁并列 `addSp`（`gameStore` 新 action）。`sp` 已在存档 `partialize` 中，**零存档版本升级**。
- Cormorant 纯数据（`ships.js` 一条 + balance.md 锚点行），登舰门槛复用既有 `canBoard`，无引擎改动——顺带验证 v0.7"新内容 = 纯数据"的达成度仍然成立。
- 抗性签名纯数据（`npcs.js` 四个 NPC 的 `defense` 字段）；MapScreen 敌情显示无需改（徽标按 `weapon.stats.damage` 计算，与抗性无关）。
- FR-2 的反查建议做成 `skills.js` 的纯函数（输入 skillKey，扫描 SHIPS/MODULES 输出解锁清单），SSR 测试可直接断言。
- 验证沿用 Node 纯函数测试惯例：spReward 缩放曲线与 elite 倍率、老档载入 sp 字段兼容、四 NPC 弱抗签名数值断言（弱抗 ≤ 其余三系 − 20）、Cormorant 登舰门槛两条路径（switchShip / buyShip 自动登舰）。

## 验收标准

- [ ] 深度 1 普通节点胜利 +200 SP 量级，elite 翻倍，随深度增长；结算浮层与 ISK 并列可见。
- [ ] 新档正常游玩（不改存档）：约 5 场内可解锁 Destroyers I，30–50 场内达成第一个 T2 门槛。
- [ ] SKILLS 页签能看到每个门槛技能解锁的具体船体/模块清单。
- [ ] 对 Guristas 选 KIN 主伤武器的 TTK 显著短于错误伤害类型；其余三派系同理（四派系签名齐备）。
- [ ] Cormorant 可购买、需 Destroyers I 登舰；15 km 磁轨 build 打其解锁深度基线 NPC 的 TTK 落在 20–60 s 带内。
- [ ] 旧存档载入：SP 与技能等级原样保留，老 NPC 新抗性即时生效（存档只存 id，天然如此）。

## 风险与开放问题

- SP 产出速率与训练价的比率是本版唯一敏感数值：过快 → 门槛失去长线意义；过慢 → 仍然不可达。基线（200/场）按"30–50 场达成首个 T2"倒推，上线后按实际节奏只调 balance.md 锚点。
- 阵亡是否给应得 SP 的一部分（如 25%）作安慰奖：倾向**不给**（保持 roguelike 损失锐度、避免"送死刷 SP"），若实测死亡螺旋过陡再回头加。
- 老 NPC 抗性收紧对错误伤害类型的玩家是隐性增强敌人：注意别把深度 1 新手局（初始 fit 是混合炮 = KIN/TH，恰好克 Guristas）推出 TTK 带——新手路径反而变顺，但要防止其他初始武器组合被卡。
