## 设计主线

1. **先地基后内容**：v0.2/v0.3 修复"无法持续游玩"（无存档）和"经济软锁"（阵亡后部分模块永久无法再获得）两个结构性问题。
2. **让选择有意义**：v0.4 的敌人原型让装配决策（射程、抗性、速度）产生对错；v0.5 的被动属性系统解锁 EVE 模块目录的大头（低槽被动件）。
3. **内容跟着渠道走**：模块/舰船的大规模补全（v0.7）放在掉落系统（v0.6）之后——roguelike 内容广度的价值来自"这局捡到了什么"，先有渠道再铺内容。

## 更远的候选（未排期）

- 地图多节点分支（Slay the Spire 式路线选择）
- 无人机系统（船体数据中 `drone_bay` / `drone_bandwidth` 已预留）
- 导弹爆炸半径/爆炸速度公式、伤害溢出穿层、敌方电容模拟
- 弹药系统、战斗中撤退机制、保险机制
