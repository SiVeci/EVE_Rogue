## 设计主线

1. **先地基后内容**：v0.2/v0.3 修复"无法持续游玩"（无存档）和"经济软锁"（阵亡后部分模块永久无法再获得）两个结构性问题。
2. **让选择有意义**：v0.4 的敌人原型让装配决策（射程、抗性、速度）产生对错；v0.5 的被动属性系统解锁 EVE 模块目录的大头（低槽被动件）。
3. **内容跟着渠道走**：模块/舰船的大规模补全（v0.7）放在掉落系统（v0.6）之后——roguelike 内容广度的价值来自"这局捡到了什么"，先有渠道再铺内容。
4. **第二周期：先修闭环，再立结构，最后扩管线**（v0.8–v0.12）：v0.8 补 SP 产出（与 v0.2/v0.3 同构的经济闭环缺口）；v0.9/v0.10 分别把"一局的形状"（分支地图、HP 持续、撤退、保险）和"模拟的完整性"（导弹公式、敌方电容、穿层、堆叠惩罚）立起来；弹药（v0.11）与无人机（v0.12）各自需要新管线（消耗品库存、多实体战斗），按管线依赖顺序收尾——两者也是"按敌情选伤害类型"同一决策语言的两个投放口。

## 更远的候选（未排期）

原候选池中的地图分支、撤退、保险（v0.9）、导弹公式/穿层/敌方电容（v0.10）、弹药（v0.11）、无人机（v0.12）已全部排期。剩余与新增：

- 巡洋舰级与 T2 船体（突击护卫）——等驱逐舰级的平衡反馈
- 多敌人遭遇 / 敌方编队（v0.12 多实体引擎落地后的自然延伸）
- Boss 节点与 run 终点结构（v0.9 段末精英位的升级）
- 蓝图掉落与工业深化（INDUSTRY 从 v0.11 弹药批产扩展到装备制造）
- NPC 主动修复与能量战扩展（电容注入器、Nosferatu——v0.10 敌方电容就位后可做）
- 装备词缀（Abyssal roll）、残骸打捞、深渊天气效应
