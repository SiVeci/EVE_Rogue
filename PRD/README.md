## 设计主线

1. **先地基后内容**：v0.2/v0.3 修复"无法持续游玩"（无存档）和"经济软锁"（阵亡后部分模块永久无法再获得）两个结构性问题。
2. **让选择有意义**：v0.4 的敌人原型让装配决策（射程、抗性、速度）产生对错；v0.5 的被动属性系统解锁 EVE 模块目录的大头（低槽被动件）。
3. **内容跟着渠道走**：模块/舰船的大规模补全（v0.7）放在掉落系统（v0.6）之后——roguelike 内容广度的价值来自"这局捡到了什么"，先有渠道再铺内容。
4. **第二周期：先修闭环，再立结构，最后扩管线**（v0.8–v0.12）：v0.8 补 SP 产出（与 v0.2/v0.3 同构的经济闭环缺口）；v0.9/v0.10 分别把"一局的形状"（分支地图、HP 持续、撤退、保险）和"模拟的完整性"（导弹公式、敌方电容、穿层、堆叠惩罚）立起来；弹药（v0.11）与无人机（v0.12）各自需要新管线（消耗品库存、多实体战斗），按管线依赖顺序收尾——两者也是"按敌情选伤害类型"同一决策语言的两个投放口。
5. **第三周期：先清引擎债，再立终点，最后扩内容**（v0.13–v0.17）：v0.13/v0.15 兑现前两周期明文记录的到期欠账（v0.12"敌侧多实体经验先在玩家侧攒"→ 敌方编队；v0.10"等观察后补"→ 被中和的反制与 NPC 主动修复）；v0.14 给 run 一个"赢"的定义（v0.9 段末 elite 顶位的正主）；v0.16 走进 v0.11 预留的蓝图数据门（Meta/T2 损失从"再赌掉落"变为"攒钱重造"）；v0.17 在全部系统反馈就位后做船级天花板抬升——纯数据收尾，与 v0.7 在第一周期的位置同构。开工前置：README 遗留项里的浏览器实机冒烟验证积压应先清（多个 deferred 决策依赖实机反馈）。

## 更远的候选（未排期）

原候选池中的多敌人编队（v0.13）、Boss 节点（v0.14）、NPC 主动修复与能量战（v0.15）、蓝图工业（v0.16）、巡洋舰与 T2 船体（v0.17）已全部排入第三周期。剩余与新增：

- 装备词缀（Abyssal roll）、残骸打捞、深渊天气效应——三者都需要"模块实例状态"（v0.2 存档预留的 version 升级门），迁移成本高，值得单独立项
- 材料/矿石轴与制造队列（v0.16 工业的第二层，等"蓝图 + ISK 重造"循环验证后）
- 战列舰级 / T3 / 派系船体（v0.17 巡洋舰反馈后）
- 敌方增援/波次与脚本化 Boss 机制（AOE、位置机变——v0.14 阶段切换原型之后）
- 中型以上无人机全系与无人机增幅模块（v0.12/v0.17 遗留的答案型模块惯例）
- 舱容 m³ 上限（v0.11 缓议）、无人机 HP 跨节点损耗（v0.12 记录在 balance.md 的候选阀门）、run 中间态持久化（v0.9 缓议）——不足以撑独立版本，按实机观察结果挂靠第三周期各版顺路交付
