# PRD v0.4 — 敌人多样性与答案型模块

## 概述

新增 3 个敌人原型（近轨爆能手 / 远程磁轨手 / 电子战护卫），`getEncounter` 按深度从敌人池抽取；同版本补上针对新敌人的"答案型"模块（MWD），并为其实现信号半径惩罚模拟。

## 背景与动机

- 当前唯一敌人 `guristas_frigate`，深度提升是纯数值放大；抗性、射程、速度等装配决策没有对错，roguelike 的"build 应对遭遇"核心循环不存在。
- `npcs.js` 数据结构与 BattleScene 的 AI 参数化（`ai.orbitRange`、武器 `type`）已为多原型铺路；玩家侧炮台命中公式（optimal/falloff + tracking vs 角速度）已实现，可直接复用到敌人侧。

## 目标

- 不同敌人迫使不同应对：贴脸敌人该风筝、风筝敌人该贴脸、减速敌人考验推进配置。
- 深度进程从"数字变大"变成"遭遇组合变化"。
- 地图节点显示敌情，玩家能在出击前针对性改装。

## 功能需求

### 敌人原型（`npcs.js` 新增 3 个）

| 原型 | 武器 | AI | 玩家的答案 |
|------|------|-----|-----------|
| **Serpentis Brawler**（近轨爆能手） | hybrid_weapon：optimal 1,500m，高 DPS，tracking 中 | orbitRange 1.0km，base_speed 350 m/s | 远程武器 + 保持距离，或硬抗互轰 |
| **Guristas Sniper**（远程磁轨手） | hybrid_weapon：optimal 15,000m，falloff 5,000m，tracking 很低 | orbitRange 15km，base_speed 220 m/s | MWD/加力贴脸，进入其 tracking 死区 |
| **Angel Webber**（电子战护卫） | missile_weapon：中等 DPS，range 9,000m | orbitRange 2.5km，携带 ewar：10km 内对玩家施加 50% 减速 | 更强推进抵消减速，或远程直接不进网 |

- **FR-1 敌人侧炮台命中**：为使用 `hybrid_weapon` 的 NPC 复用玩家侧命中公式（对玩家的 `sig_radius` 与相对角速度计算）。玩家高速小角速度绕轨可以真实规避 Sniper 的炮。
- **FR-2 敌人侧 EWAR**：NPC `ewar` 数据驱动对玩家的减速 debuff（与玩家 web 敌人使用同一套速度修正管线）。
- **FR-3 遭遇池**：`getEncounter(depth, nodeType)` 改为从深度解锁的池中随机抽取。建议解锁表：深度 1–2 仅原版 Guristas；3+ 加入 Brawler；5+ 加入 Sniper；7+ 加入 Webber。数值缩放公式（`1 + 0.15×(depth−1)`，elite ×1.5）不变。
- **FR-4 敌情显示**：MapScreen 节点上显示本次遭遇的敌人名称与武器类型（进入节点前已确定，非进入后才 roll），玩家可据此回站改装。

### 答案型模块

- **FR-5 1MN MWD I**（propulsion / mid）：速度倍率 ~5.0，但开启期间玩家 `sig_radius` ×5、电容消耗高。这是对 Sniper 的核心答案，也引入"开 MWD 冲锋时更容易被打中"的权衡。
- **FR-6 信号半径惩罚模拟**：BattleScene 支持模块对玩家有效 `sig_radius` 的运行时倍率；敌人侧炮台命中公式（FR-1）消费该值。此机制同时为 v0.6+ 的导弹爆炸公式与护盾扩展的信号惩罚打底。

## 非目标

- 多敌人同屏（1v1 保持不变；多目标锁定/UI 是大工程，另立版本）。
- 敌方电容模拟、玩家战斗中撤退。
- 敌人使用修理/推进模块。

## 技术方案要点

- BattleScene 敌武器处理从"missile 专用分支"泛化为按 `weapon.type` 分支（与玩家模块处理循环同构）；命中公式抽成两侧共用的纯函数。
- 敌情预生成：App.jsx 在进入 map 视图时生成两个节点的遭遇（或以 `depth+nodeType` 为种子），传给 MapScreen 与 BattleScene，保证"看到的就是打到的"。
- 受影响文件：`src/data/npcs.js`、`src/components/BattleScene.jsx`（敌人侧武器/EWAR、sig 倍率）、`src/components/MapScreen.jsx`、`src/App.jsx`、`src/data/modules.js`（MWD）。

## 验收标准

- [ ] 深度 3+ 能遇到 Brawler，5+ 能遇到 Sniper，7+ 能遇到 Webber（可临时调高概率验证）。
- [ ] 对 Sniper：静止挨打命中率高；1km 高速绕轨时其命中率显著下降（观察战斗日志/伤害频率）。
- [ ] 开 MWD 冲向 Sniper 期间，被命中率明显高于关 MWD 绕轨时（sig 惩罚生效）。
- [ ] 被 Webber 减速时玩家实际速度减半；用 meta 加力可部分抵消。
- [ ] MapScreen 显示的敌人与进入后遭遇一致。
- [ ] 原版 Guristas 战斗行为与 v0.3 完全一致（回归）。

## 风险与开放问题

- 三原型 + 深度缩放的平衡需要游玩迭代；先保证"应对方式正确时明显更轻松"，数值精调放宽容忍度。
- Webber 减速与玩家 web 敌人共用管线时注意双向 debuff 的叠乘顺序（建议统一为乘法叠加）。
