# PRD v0.10 — 战斗模拟深化

## 概述

偿还 CLAUDE.md"当前缺口"中列明的四笔引擎级欠账：导弹爆炸半径/爆炸速度公式、敌方电容模拟（并投放玩家能量中和模块作为其对手盘）、伤害穿层、重复 `mult` 修正的堆叠惩罚。主题：把**速度/信号、电容、堆叠收益**这几条 EVE 经典对抗轴全部接入模拟，让装配选择的对错面更完整。本版几乎全部工作在 `src/lib/` 纯函数层，UI 改动极小。

## 背景与动机

- **导弹射程内必中且全额伤害**：速度 tank 与小信号对导弹系敌人（Guristas 火箭、Angel 双船）完全无效，MWD 的信号惩罚对导弹也不存在——装配决策整整少一个维度，Atron（拿生存换速度的船）的设计意图被导弹敌人单方面绕过。模块数据里 `explosion_radius` / `explosion_velocity` 字段自 v0.1 起就存在，一直未接线。
- **敌方是电容永动机**：玩家侧电容是战斗的核心资源，敌方却没有这个约束；能量战（中和）这条 EVE 标志性 EWAR 轴没有受体，永远无法投放。
- **溢出伤害浪费**：破层 volley 的溢出被丢弃，高 alpha 武器（磁轨）实际强度低于面板，且鼓励"卡层"的怪异手感。
- **堆叠无惩罚**：v0.5 已标注"若堆同名增幅成为支配策略再回头"；v0.7 的 T2 增幅（×1.13）入场后，双/三增幅低槽已接近无脑最优，临界点到了。

## 目标

- 速度与信号成为对导弹的真实防御轴：快船少吃导弹伤害，开 MWD 的信号 bloom 把减免吐回去（与炮台侧的既有语义对称）。
- 电容压制成为可行 build 方向：中和器把敌方电容打空 → 敌方武器/EWAR 停转。
- 面板与实际伤害的偏差可解释：穿层后 alpha 不再浪费；堆叠惩罚后第 3 件同属性增幅明显不值，低槽出现真实的机会成本。

## 功能需求

- **FR-1 导弹伤害公式**：`damageFactor = min(1, sig/expR, ((sig/expR) · (expV/v_target))^drf)`（EVE 简化版，drf 先取 0.5）。玩家导弹模块的既有 `explosion_radius` / `explosion_velocity` 字段直接启用；NPC 导弹武器补齐这两个字段，数值按"对静止护卫 factor ≈ 1"标定——现有 TTK 不漂移，公式只在目标高速/小信号时产生差异。玩家的 MWD `sig_multiplier` 自然参与目标信号（信号惩罚从此对导弹也成立）。
- **FR-2 敌方电容**：NPC 数据增加 `cap_capacity` / `cap_recharge`，武器与 `ewar` 每次激活消耗电容，回充复用 `getCapacitorRecharge`；电容不足则该次激活跳过（与玩家模块语义一致）。基线标定为**无外力时永远电容稳定**——行为与现状零差异，它只是中和的受体。未定义电容字段的 NPC 回退为无限电容（数据逐个补，向后兼容）。
- **FR-3 能量中和模块**：新模块类型 `energy_neut`（高槽，Small Energy Neutralizer I 市场 T1 + Meta 变体入掉落表），周期性烧掉目标电容、自身电容开销高；BattleScene 模块循环加一个分支——**本版唯一允许的新类型分支**。敌方电容被压空 → 武器与 web 停转，回充过激活线后恢复。
- **FR-4 伤害穿层**：`applyDamage` 改为溢出伤害带着下一层的抗性继续结算（单 volley 穿一层封顶，现有 alpha 量级穿两层的场景不存在）；玩家与 NPC 对称生效。
- **FR-5 堆叠惩罚**：`getEffectiveStats` 中同一 stat 的 `mult` 修正按强度降序应用 EVE 惩罚序列（100% / 86.9% / 57.1% / 28.3% / …）；技能倍率与 `add` 修正不受罚。惩罚表与豁免规则写入 balance.md，CLAUDE.md 同步删掉对应"当前缺口"条目。

## 非目标

- 玩家侧被中和的反制（电容注入器/电池）与 NPC 主动使用中和——先看玩家中和 build 的强度再定。
- 导弹弹种（v0.11 弹药系统的载体）。
- NPC 主动修复模块（电容就位后成为可能，入候选池）。
- drf 精确拟合 EVE 原公式、导弹飞行时间/弹道模拟（仍是即时命中判定）。

## 技术方案要点

- 公式全部落在纯函数层：`missileDamageFactor` 进 `src/lib/combat.js`，穿层改造 `applyDamage` 本体，堆叠惩罚进 `src/lib/shipStats.js` 的 mult 合并处——两侧共用、可单测，这是本版主体。
- **平衡回归面**：穿层普遍加快互杀（双向），堆叠惩罚定向削多增幅玩家 build，方向相反——TTK 带（20–60 s）要在两者叠加后复核，本版验收允许二轮修数。
- **旧档兼容**：堆叠惩罚会降低 Co-Processor（CPU `mult`）等叠加 fit 的容量上限，已装超限的旧 fit 按现有"只在装/卸时校验"的语义继续存在、不强制拆解（与 `unfitModule` 的守卫逻辑一致），测试覆盖"超限旧档可载入、卸下后装不回去"。
- 中和的敌方停转判定复用 FR-2 的"电容不足跳过激活"，不需要额外 AI 状态。
- 验证重心是 Node 数值测试：factor 曲线（静止=1、Atron 400 m/s 显著<1、web 后回升）、电容稳定标定（全 NPC 无中和时输出节奏与现状逐拍一致）、穿层前后 TTK 对照、堆叠序列断言（第 2/3 件 ×0.869/×0.571）。

## 验收标准

- [ ] 400 m/s 的 Atron 吃火箭伤害显著低于被 web 后的同船；开 MWD 期间（sig bloom）导弹伤害回到满额。
- [ ] 静止目标的导弹实际伤害与面板一致（factor = 1 标定成立），现有导弹 NPC 对静止玩家的 TTK 不漂移。
- [ ] 装中和器压制 Serpentis Scout：电容压空后其 web 与炮停转，停止中和后随回充恢复。
- [ ] 磁轨 alpha 破盾的溢出以装甲抗性进入装甲层，同 fit 总 TTK 短于现状。
- [ ] 第 2 件磁场稳定器实际收益 ≈ 0.869 倍、第 3 件 ≈ 0.571 倍（数值断言）；单件 fit 与现状完全一致。
- [ ] 全部 8 个既有 NPC 补电容字段后，无中和情况下行为与现状零差异（回归测试）。
- [ ] BattleScene 除 `energy_neut` 外无新增类型特判分支（v0.7 管线审计惯例复跑）。

## 风险与开放问题

- 中和在 1v1 语境下是"把对面关机"的单点压制，可能过强：安全阀备选——压空后敌方武器降频而非全停、或提高中和自身电容开销到接近自残。实测（Serpentis Scout / Angel Webber 两个 EWAR 受体）后定。
- drf = 0.5 决定快船减伤的手感幅度，Atron 与 Angel Hunter 两个速度标杆先测；过强则 drf 上调（减伤变浅）。
- 堆叠惩罚是对现存玩家 build 的定向削弱，发布说明需明示；若双增幅 build 从"最优"跌到"不可用"说明惩罚序列过陡（EVE 序列理论上不会，但我们的槽位远少于 EVE）。
- 四项同版落地的回归面靠"全部落纯函数 + 逐项对照测试"控制；若工期紧，FR-4/FR-5（互相独立）可拆出为 v0.10.1。
