# PRD v0.12 — 无人机系统

## 概述

启用船体数据里预留已久的 `drone_bay` / `drone_bandwidth`：无人机作为**可损耗的伴飞作战单位**（发射/回收、自动接战、会被打掉、掉了要花钱补），配套无人机技能、4 型伤害类型无人机与首批无人机船体（Tristan 护卫 + Algos 驱逐）。这是战斗引擎自 MVP 以来第一次从"双实体"走向多实体，也是 Gallente 无人机 build 路线的解锁。

## 背景与动机

- `drone_bay` / `drone_bandwidth` 从 v0.1 起就在船体数据上（Incursus 5/5、Kestrel 0/0），始终未启用；EVE 玩家对 Gallente 无人机路线有天然预期，v0.7 因此明确把 Tristan 排除在船体扩充之外。
- 无人机是**可损失的战斗资产**：被击落就没了、要花 ISK 补——与 roguelike 的损失张力同构，比"再加一种武器"更契合本作主题。
- 4 型伤害类型无人机与 v0.11 弹药是同一决策语言（按敌情选伤害类型）的两个投放口，玩家已被教育过，学习成本为零。
- 依赖：v0.11 的堆叠计数管线（无人机机库存量按件计数，复用 cargo 语义）；v0.10 的导弹公式让无人机的小信号/高速天然获得对导弹的生存力（引擎红利，无额外工作）。

## 功能需求

- **FR-1 无人机物品**：新数据文件 `src/data/drones.js`：轻型侦察无人机 4 型——Hobgoblin（TH）/ Hornet（KIN）/ Warrior（EXP）/ Acolyte（EM），各 5 m³ / 5 Mbit；自身 HP 小、速度 ~600 m/s 级、单机 DPS ≈ 一门 T1 炮的 60–70%。T1 市场有售，Meta/T2 变体走掉落（品级管线沿用）。
- **FR-2 挂舱与带宽**：FittingWindow 新增无人机挂舱区：装载总量受 `drone_bay`（m³）限制；同时在空数量 = min(带宽允许数, drones 技能等级)。Incursus 5/5 = 恒 1 架（风味保留），Kestrel 0/0 = 不能装载。
- **FR-3 战斗行为**：LAUNCH / RECALL 一键放收（全队指令，无单机微操）；在空无人机自动接战当前敌人：近轨环绕、按周期输出，伤害走 `applyDamage` 全套抗性/分层结算。敌方每隔数秒有概率将火力转向随机一架在空无人机数拍（命中判定复用其武器全套公式——小 sig 高速的无人机天然难被大炮打中）；**被击毁的无人机永久损失**。战斗结束（胜利或撤退）自动回收存活个体；玩家阵亡则舱内与在空无人机全损。
- **FR-4 无人机技能**：新技能 `drones`：等级 N = 最多同时 N 架在空（与带宽取小）。这是门槛与数值的混合形态，也是 v0.8 之后 SP 的又一长线出口（练满 5 架 = 15,000 SP，与 T2 门槛同带）。
- **FR-5 无人机船体**：Tristan（Gallente 护卫，bay 25 / bw 25 → 满编 5 架轻型，炮位少、主输出在无人机）；Algos（Gallente 驱逐，`requiredSkills: { destroyers: 1, drones: 1 }`，炮 + 无人机混合平台）。数值按 balance.md 各船级锚点填表，并新增"无人机 DPS 计入船体 DPS 预算"的分账原则（见风险）。
- **FR-6 敌情扩展（可选，可滑落为内容补丁）**：新 NPC 派系 Rogue Drones（自主无人机风味的单实体 NPC，掉无人机与相关件）——纯数据，管线无新需求。

## 非目标

- 敌方使用无人机、多敌人编队（敌侧保持单实体；多实体引擎经验先在玩家侧攒）。
- 中型/重型/哨戒无人机、修理无人机、EWAR 无人机。
- 单机微操（集火、分散、驻守指令）与无人机独立命令条。
- 无人机增幅模块（伤害放大器等）——等 build 成型后按答案型模块惯例补。
- 无人机的弹药消耗（无人机不吃 v0.11 弹药）。

## 技术方案要点

- **引擎主体**：BattleScene 的 refs 模式从"双实体"扩展为"双主体 + 玩家无人机数组"（每架：pos/vel/hp/攻击计时），`PhysicsUpdater` 帧循环内迭代；无人机**不进 `modulesStateRef`**（不是模块），LAUNCH/RECALL 走 `commandRef` 旁的独立命令入口。这是本版最大工程量与回归面所在。
- 敌方转火：敌武器结算时按概率把本拍目标切为随机在空无人机，沿用其 weapon 的既有公式（炮的 tracking 对小目标天然吃瘪、导弹走 v0.10 factor）——不为无人机写任何专用命中逻辑。
- 数据/存档：`drones.js` 静态数据；store 增加机库无人机存量（复用 v0.11 计数语义）与 activeShip 挂舱清单（随船损毁，与模块同规则）；`partialize`/`merge` 增量 key 缺省回填，预计**零版本升级**。
- 3D 表现：极简 mesh（小三角/点精灵）+ 环绕目标的轨道动画，沿用 1 unit = 1 km 与夸张缩放惯例；HUD 增加无人机编队状态条（在空数 / HP 概览 / LAUNCH-RECALL）。
- 实施建议：先落"1 架无人机"的最小纵切片（发射-接战-被打-回收全流程），再放开到 5 架编队——多实体的回归风险用切片顺序控制。
- 验证：Node 测试覆盖带宽/技能/舱容三重限制矩阵、损耗结算（击毁扣存量、回收保留、阵亡全损）、无人机 DPS 对四派系弱抗的 TTK 差；既有全部战斗回归（无人机数组为空时逐拍行为与现状一致）。

## 验收标准

- [ ] Tristan 放出 5 架 Hobgoblin：持续输出、敌方偶尔转火并可能击落；胜利后存活个体回舱、损失个体从存量扣除。
- [ ] drones 技能 2 级同船最多 2 架在空；Incursus 恒 1 架；Kestrel 无法装载（三重限制矩阵成立）。
- [ ] 对 Sansha（弱 EM）带 Acolyte 的 TTK 显著短于 Hornet——伤害类型决策与弹药同构。
- [ ] 玩家阵亡：舱内与在空无人机全损、机库存量保留；补充购买流与模块一致。
- [ ] v0.9 撤退时在空无人机自动回收，对齐读条不因无人机中断。
- [ ] 无人机全程不产生新的模块 type 特判分支；无人机数组为空时既有战斗行为零回归。

## 风险与开放问题

- **多实体是引擎质变**：帧循环、命中结算、HUD 全部长出数组维度，回归面覆盖全部既有战斗——用"1 架切片先行"与空数组零回归测试双保险；若实施中发现 refs 模式撑不住，暂停加实体数并回头补引擎，不硬堆。
- **敌方转火概率是无人机损耗率的总阀门**：太低 → 无人机是无代价的白嫖 DPS；太高 → 一场清空存量。起步锚点"平均每 2–3 场损失 1 架"，配合单价让无人机损耗 ≈ 弹药消耗的 2–3 倍（都入 balance.md）。
- **DPS 分账**：Tristan 满编无人机 + 剩余炮位的总 DPS 必须受预算约束（balance.md 红线：不超过同级纯炮船的 110%），否则无人机船直接淘汰纯炮船。
- Rogue Drones 派系（FR-6）与主体工程解耦，工期紧时最先滑落。
