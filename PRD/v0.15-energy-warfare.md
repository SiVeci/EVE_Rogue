# PRD v0.15 — 能量战扩展与 NPC 主动修复

## 概述

把 v0.10 建成的电容系统从"玩家单向压制"补成双向攻防：玩家侧新增被中和的反制（电容注入器 + 电容电池）与吸取型武器（Nosferatu）；NPC 侧新增主动修复模块（受自身电容 floor 门控——中和成为破修手段）与首批会用中和的 NPC。电容从资源条升格为攻防轴。

## 背景与动机

- CLAUDE.md"当前缺口"明文记录两笔：玩家没有被中和的反制（cap injector/battery）、NPC 没有主动中和与主动修复——都标注"v0.10 敌方电容就位后可做，等观察玩家 neut build 强度"。两个观察期版本（v0.11/v0.12）已过，到期清偿。
- v0.10 给了全体 NPC 电容预算与激活地板，但地板目前只有玩家 neut 一个施压者、NPC 电容只有武器/EWAR 两个消费者——基础设施利用率一半。NPC 主动修复是地板机制的天然第三消费者：**修复受电容门控，中和压地板 = 破修**，玩家 neut build 从"让敌人闭嘴"升级为"让敌人不能回血"，决策闭环完整。
- Sansha 派系在 EVE 里是能量战招牌，但本作两个 Sansha NPC 只掉中和模块、自己不用——风味债顺路清偿。
- 依赖：v0.10（电容/地板机制）、v0.11（注入剂复用弹药 cargo 管线）。与 v0.13/v0.14 无硬依赖，可并行或提前。

## 功能需求

- **FR-1 电容注入器**：mid 槽主动模块 Small Capacitor Booster I（T1 市场 / Meta 掉落），消耗**电容注入剂**（Cap Booster 25/50 两规格）：激活瞬时注入对应电容量，装填周期 12s（复用 v0.11 装填读条）。注入剂进 `AMMO` 数据空间（family `cap_booster_s`），走 cargo 堆叠库存、市场按手购买、INDUSTRY 蓝图批产——**全管线复用，零新库存概念**。
- **FR-2 电容电池**：low 槽被动件 Small Cap Battery I/II（T1/T2）：`modifiers` 加电容容量，并新增**中和抗性**属性（进入方 neut 量按 % 折减——EVE 原生机制）；纯数据被动件 + `applyDamage` 侧一处折减读取。
- **FR-3 Nosferatu**：high 槽 Small Nosferatu I（T1 市场）：按周期吸取目标电容转入自身（量级 ≈ 同级 neut 的 60%，自身 cap_use 0——nos 是产能型），受"目标电容高于自身"的 EVE 式条件门控。模块 type 新增 `energy_nos` 分支。
- **FR-4 NPC 主动修复**：NPC 数据新增可选 `repair` 块（`layer`/`repair_amount`/`activation_time`/`cap_use`）：2 个原型先行——Serpentis Brawler 变体修甲、Guristas 变体修盾（以新深层 NPC 或既有 NPC 的深层变体形式入 `ENCOUNTER_UNLOCKS`）。修复照常受 `npcActivationFloor` 门控：**电容被压穿地板时修复停转**。红线入 balance.md：修复量 ≤ 合理 T1 fit DPS 的 40%（未破修时 TTK 拉长但必然可击杀，破修后 TTK 恢复正常带）。
- **FR-5 NPC 中和**：Sansha 两 NPC 的深层变体携带 `neut` 块（复用 `ewar` 块的 shape：optimal/activation_time/cap_use/neut_amount）：周期性烧玩家电容——玩家主动模块（修复/MWD/web/武器）开始经历"电容被打空"的另一半体验，注入器（FR-1）是其答案。强度不随深度缩放（EWAR 惯例）。
- **FR-6 战斗可读性**：HUD 电容条增加被中和/被吸取的瞬时反馈（数值跳变 + 日志行）；敌方修复启动/停转（破修）打日志——中和 build 的正反馈必须可见。

## 非目标

- 远程电容传输（无友军目标）。
- 中型/大型注入器与电池（船级到货再说，v0.17 顺路）。
- 全 NPC 铺开主动修复/中和（2+2 原型先行，观察后走内容补丁扩量）。
- NPC 使用注入剂（NPC 电容仍纯靠 recharge——对称性让位于 AI 复杂度，同"NPC 不消耗弹药"惯例）。
- 电容战 EWAR 大类（能量吸取无人机等）。

## 技术方案要点

- 注入剂进 `src/data/ammo.js`（新 family，条目带 `charge: { cap_bonus }`），`merge` 的 `AMMO[id]` 校验、`buyAmmo`/`sellAmmo`、蓝图路由**天然全通**——这是把注入剂定义为弹药而非新物类的全部理由。注入器武器位走 `modulesStateRef` 既有 `{ ammoId, reload }` 形态（mid 槽首次带弹：`activeShip.ammo` 的"高槽平行数组"约定需扩展为高+中槽，或注入器单独挂 ammo 字段——**实施计划里定夺，倾向后者**，避免动 v0.11 的对齐不变量）。
- 模块 type 分支：+2（`cap_booster`、`energy_nos`）——七分支审计惯例更新为九，PRD 明示这是授权增长。
- NPC `repair`/`neut` 块镜像 `ewar` 块的处理位置（敌成员帧循环内、地板门控前置）；v0.13 多敌下逐成员独立运行（若本版先于 v0.13 实施则单敌照常，两版无耦合）。
- 中和抗性折减点在 neut 结算处统一（玩家 neut 对 NPC、NPC neut 对玩家同一函数）：`combat.js` 新纯函数 `applyNeut(amount, targetNeutResist)`。
- balance.md 新增"能量战"一节：注入器抵抗 1 个 T1 neut 有余、2 个必然被压穿（数值锚点：25 型注入速率 ≈ 1.2× T1 neut 速率、50 型装填更久总量同带）；NPC 修复红线与破修 TTK 对照表。
- 验证：Node 测试覆盖注入/吸取/抗性折减矩阵、NPC 修复的地板停转与恢复时序、"1 neut vs 注入器"与"2 neut vs 注入器"的收敛性、无新模块时全量零回归。

## 验收标准

- [ ] 被 Sansha neut 变体压空电容后：打注入剂恢复模块运转；注入剂耗尽则回到被压制状态（消耗品张力成立）。
- [ ] 对 NPC 修甲变体：不带 neut 的 fit TTK 明显拉长但可击杀；带 1 neut 压穿地板后修复停转、日志播报"修复停转"、TTK 回落正常带。
- [ ] Nosferatu：自身电容低于目标时每周期净得电容；高于目标时不吸取（EVE 条件门控成立）。
- [ ] Cap Battery II 装备后，同一 NPC neut 的每周期实际烧蚀量按抗性折减；stacking penalty 对电池的 add 容量不适用（add 豁免回归）。
- [ ] 注入剂走市场按手购买、INDUSTRY 75% 折价批产、阵亡 cargo 保留——与弹药规则处处一致。
- [ ] 不装任何新模块、不遇新 NPC 变体时，v0.12 全量战斗行为零回归。

## 风险与开放问题

- **"不可击杀 NPC"是本版最大红线**：修复量超标 = 软锁一场战斗（只能撤退）。40% DPS 红线 + Node 模拟先行校准；调参杆固定为 `repair_amount` 单杆。
- **注入器让 neut 对抗玩家失效的风险**：若注入剂便宜到无限量携带，FR-5 的 NPC 中和形同虚设——注入剂单价定在弹药带的 5–8 倍（一场被 neut 的战斗注入成本 ≈ 赏金 20–30%），让"带注入器"是应对而非默认。
- mid 槽带弹打破"只有高槽有 ammo"的 v0.11 对齐不变量——实施计划必须先定存档形态（倾向注入器 ammo 单独挂字段，`merge` 增量校验，零版本升级；若做不到干净则接受 v4 版本升级）。
- NPC 修复/中和变体占用 `ENCOUNTER_UNLOCKS` 深层权重位：与 v0.13 编队、v0.14 Boss 的深层内容密度需要一次统筹（三版实施顺序上谁先谁后都成立，但深度表要一起排）。
