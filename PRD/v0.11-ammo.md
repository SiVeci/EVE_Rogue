# PRD v0.11 — 弹药系统

## 概述

武器从"自带伤害"改为"发射弹药"：混合炮装药在射程与伤害间取舍，导弹弹头自由选择伤害类型；弹药是可堆叠的消耗品（**新库存管线**），市场廉价购买、工业批量制造、战斗持续消耗。这是"看敌情 → 配弹 → 出击"决策环的最后一块，也是 ISK 的第一个常态化出口。

## 背景与动机

- v0.7/v0.8 建成了完整的四派系弱抗签名，但玩家换伤害类型目前只能**换整支武器**——炮系伤害类型甚至锁死 KIN/TH，对 Sansha（弱 EM）永远打不出正确答案。"按敌情配置"止步于防御端（换抗镀层），主动端缺一半。
- ISK 只有装备/船体的一次性购买出口，中后期结余堆积、赏金边际价值走低；弹药是 EVE 式的每场常态 ISK 汇。
- INDUSTRY 页签自 MVP 起是存根（两张写死蓝图 = 打折买模块），弹药批产是它第一个真实用例。
- 依赖：v0.8（四派系弱抗齐备，换弹才有答案）、v0.10（导弹公式落地，弹头才有爆炸参数可携带）。

## 功能需求

- **FR-1 弹药物品与堆叠库存**：新数据文件 `src/data/ammo.js`（id、适配武器族、单价、修正/伤害定义）；store 新增 `cargo: { [ammoId]: qty }` 堆叠库存——与模块 `inventory`（逐件数组）分离，不动既有管线。市场新增 AMMO 货架，按 100 发一手交易。
- **FR-2 混合炮装药（射程/伤害轴）**：3 档，伤害类型维持 KIN/TH：反物质（伤害 ×1.2、optimal ×0.6）/ 标准（×1.0 基准，行为等于现状面板）/ 铱弹（伤害 ×0.85、optimal ×1.6）。修正乘在武器基础面板上，falloff 不动。
- **FR-3 导弹弹头（伤害类型轴）**：4 型同威力弹头：Mjolnir（EM）/ Inferno（TH）/ Scourge（KIN）/ Nova（EXP）——把武器伤害对象替换为单一类型、总量守恒；`explosion_radius` / `explosion_velocity` 随弹头携带（成为 v0.10 公式的参数源）。
- **FR-4 装填与消耗**：FittingWindow 为每门已装武器指定弹药；开火按周期消耗 1 发；打空自动停火并在 HUD 提示；战斗中换弹 = 10 秒装填读条（期间该武器离线）。带弹量本版不设舱容——出击自动携带机库 cargo 全部存量。
- **FR-5 获取渠道**：市场 T1 弹药常备（单价 20–80 ISK/发，锚点：一场典型战斗的弹药净消耗 ≈ 赏金的 5–15%，入 balance.md）；NPC 掉落表补弹药束（50–100 发/束，走既有 lootTable 管线）；INDUSTRY 弹药蓝图批产（单发成本 ≈ 市场 75%，沿用既有折价惯例）。
- **FR-6 强制装弹与保底**：武器必须指定弹药才能开火；新档初始 cargo 附赠足量标准装药与 KIN 弹头（新手第一课不能是"忘带子弹"）；"免费 Incursus"保底条款同步保底一手标准弹药——软锁不可能性延伸到弹药维度。旧档升级时同样补赠并给已装武器自动指定默认弹。

## 非目标

- T2/派系弹种（Javelin/Rage 式第二层取舍轴）。
- 舱容/体积系统（cargo m³ 上限）——先验证核心循环，限量带弹的紧张感留给后续。
- 激光晶体（尚无激光武器族）、弹药打捞。
- NPC 消耗弹药（敌方仍按面板输出）。
- 出击中途补给（弹药只在站内进出 cargo）。

## 技术方案要点

- 弹药修正的应用点在 `getEffectiveStats` **之后**、开火结算之前：`combat.js` 新纯函数 `applyAmmoToWeapon(weaponStats, ammo)`，面板显示与开火结算共用，保证所见即所得。
- 武器实例态挂在 BattleScene 既有 `modulesStateRef` 上（每门武器：ammoId、剩余量、装填计时）；换弹读条复用现有 cooldown 计时模式。**弹药不是模块，不新增模块 type 分支**——管线审计惯例继续适用。
- 存档：**版本 +1（v3）**。`partialize` 增加 `cargo`（id→qty）与 activeShip 每武器的 ammo 指定；`migrate` 对 v2 档补赠初始弹药并自动指定默认弹；`merge` 对未知弹药 id 沿用"静默丢弃"容错。
- UI：MarketWindow 弹药区（按手）；FittingWindow 武器行加弹药选择器（只列适配族）+ 出击前余弹可见（UNDOCK 侧的装弹状态提示）；BattleScene HUD 模块按钮显示余弹数与装填读条。
- INDUSTRY：blueprints 数据加弹药蓝图（批量产出进 cargo）——顺带把 blueprints 改为从静态数据文件读取（为将来蓝图掉落铺路，本版不做掉落）。
- 验证：Node 测试覆盖 applyAmmoToWeapon 修正矩阵、消耗/打空/换弹时序（模拟帧推进）、v2→v3 迁移（补赠+自动指定+无软锁）、掉落表含弹药束后的期望校准复跑。

## 验收标准

- [ ] 对 Sansha（弱 EM）装 Mjolnir 的 TTK 显著短于 Scourge；四型弹头对无抗目标总威力一致。
- [ ] 反物质装药 DPS 提升但 optimal 缩至 0.6 倍；MapScreen 看到 15 km 狙击手后换铱弹是正确应对（可测的 TTK 差）。
- [ ] 打空弹药武器停火并提示；战斗中换弹 10 秒后以新弹继续；标准弹装填的武器行为与 v0.10 现状完全一致（基准不漂移）。
- [ ] 深度 3 巡逻一场的弹药净消耗落在赏金 5–15% 带内；INDUSTRY 批产单发成本 ≈ 市场 75%。
- [ ] v2 旧档升级：cargo 获赠、已装武器自动指定默认弹、无软锁；重置进度后新档开箱即战。
- [ ] 阵亡：机库 cargo 保留（弹药是机库物资，与模块库存同规则）；随船损失的只有船体与已装模块，规则不变。

## 风险与开放问题

- **"必须装弹"新增一种战败原因**（带错/带少）：初始附赠 + 保底 + UNDOCK 前状态提示三重缓解；若实测仍高频翻车，回退方案是"无弹时以 ×0.5 面板降级开火"（EVE 味道差，但保底手感稳）。
- 单价 × 消耗速度是新的经济耦合点：过贵 → 玩家退回只用送的标准弹（系统白做）；过贱 → ISK 汇不成立。锚点区间先行，允许一轮修价。
- 每周期 1 发的消耗粒度是抽象化取舍：若"数子弹"在 HUD 上太琐碎，改为按激活整包扣（如 1 手/次）——只动结算粒度，不动经济锚点。
- 弹药选择器把 FittingWindow 的信息密度再抬一档：v0.7 已为 8 槽做过自适应，本版需要复查武器行的横向空间。
