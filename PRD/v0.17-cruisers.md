# PRD v0.17 — 巡洋舰级与 T2 船体

## 概述

第三船级与船体品级轴：**3 艘巡洋舰**（Thorax 近战炮 / Caracal 导弹 / Vexor 无人机）配套中型武器线与中型无人机；**2 艘 T2 突击护卫**（Enyo/Harpy）作为"小而精"的平行终局路线。这是 v0.7 式的内容大扩充，前提是驱逐级与第三周期各系统（编队、Boss、能量战）的平衡反馈已就位——本版为纯数据版本，引擎预期零改动。

## 背景与动机

- 候选池原文："巡洋舰级与 T2 船体（突击护卫）——等驱逐舰级的平衡反馈"。第三周期前四版（编队、Boss、能量战、工业）都在给深层增加压力与出口，船体天花板停在驱逐级会让深层（d15+）的成长曲线只剩模块/无人机品级一根轴。
- v0.14 的 Boss 已在 NPC 侧立了巡洋舰形状（大 sig / 慢速 / 高 HP），玩家侧巡洋舰有敌侧参照可对表；v0.13 编队让"一打多"成为常态，巡洋舰的"火力平台"定位（打得多、躲不掉）有了真实语境。
- T2 船体与巡洋舰是**同一终局预算的两种答案**：巡洋舰 = 更大的船（贵、慢、多槽），突击护卫 = 更精的小船（贵、快、双抗加成）——EVE 的经典分歧，也检验 balance.md 锚点体系能否表达"横向而非纵向"的高端。
- SP 出口再加一档：`cruisers`/`assault_frigates` 两个新 gate skill 各 15,000 SP 级门槛，与 T2 武器门槛同带，v0.8 建的 SP 经济到深层不断档。

## 功能需求

- **FR-1 巡洋舰船体与技能**：新 gate skill `cruisers`（`requiredSkills: { cruisers: 1 }`，Vexor 加 `drones: 3`）。三舰锚点（入 balance.md 新船级表）：slot 总数 15–16、EHP ≈ 护卫 ×2.5–3（3,000–3,900）、速度 180–210 m/s、agility 6–8s、sig 105–125m、价格 550,000–650,000（≈ 驱逐 ×3）。Thorax（5/4/6，近战 blaster 平台）/ Caracal（5/5/5，导弹平台）/ Vexor（4/4/6 + bay 75 m³ / bw 50 Mbit——中型 ×2 + 轻型 ×4 或轻型满编成立）。
- **FR-2 中型武器线**：中型混合炮 3 型（Electron/Ion/Neutron Blaster、Rail 各口径按 v0.7 小型线的射程/伤害梯度等比放大 ≈ ×2.2 伤害、×1.8 射程、PG/CPU ×3）+ 重型导弹发射器 2 型；全线 T1/Meta/T2 三档（品级阶梯 100/110/120% 惯例）。**弹药 family 扩展**：`hybrid_m` 3 档装药、`heavy_missile` 4 型弹头——数据复用 v0.11 全部 shape，`AMMO_LOT`/定价按单发威力等比放大（中型弹单价 ≈ 小型 ×2.5）。中型武器 `requiredSkills`: `medium_hybrid_turret`/`heavy_missiles` 新 gate skills（T1 需 1 级，T2 需 5 级——镜像小型线）。
- **FR-3 中型无人机**：Hammerhead（TH）等 4 型中型侦察无人机：10 m³ / 10 Mbit、单机 DPS ≈ 轻型 ×2、速度更慢（400 m/s 级）、HP ×2.5；Meta/T2 变体走掉落（惯例）。`maxDronesInSpace` 的 `bandwidth/5` 除数改为按无人机自身 `bandwidth` 求和判定（Vexor 50 Mbit = 中 ×2 + 轻 ×4 等混编成立）——`src/lib/drones.js` 三个 helper 是本版唯一的代码改动点。
- **FR-4 T2 突击护卫**：Enyo（blaster，4/2/4）/ Harpy（rail，4/4/2）：`requiredSkills: { assault_frigates: 1 }`（新 gate skill，其解锁自身要求玩家已有高级小炮技能的语境由训练成本表达）；护卫级机动（350–380 m/s、agility 3s 带、sig 40m 带）+ 驱逐级火力预算 90% + 天生双抗加成（`defense` 基础抗性直接写高 15–20 点——船体自带，非模块，不入 stacking penalty）；价格 400,000 级（贵过驱逐、逼近巡洋）。
- **FR-5 深层 NPC 巡洋舰**：既有五派系各出 1 个巡洋舰级 NPC（d13+ 解锁入 `ENCOUNTER_UNLOCKS`，对表 v0.14 Boss 带的 0.5–0.6 倍强度），编队中可任主位——深层遭遇的敌我船级同步升级。可滑落为内容补丁（同 v0.12 FR-6 惯例）。
- **FR-6 UI 承载复查**：FittingWindow 16 槽布局、HUD 模块架 5+ 高槽、市场/机库列表的船级分组——v0.7 为 8 槽做过一轮自适应，本版按 16 槽再验一轮。

## 非目标

- 战列舰及以上船级、T3 船体、派系/海军版船体。
- 重型/哨戒/修理/EWAR 无人机（中型侦察 4 型是本版上限）。
- 中型以上的注入器/电池之外的新模块大类（既有模块目录的中型化按需少量补充，不做全目录翻倍）。
- 舰种平衡性大改（护卫/驱逐既有数值不动——新船级向上兼容，不回调旧内容，balance.md 总纲惯例）。
- 巡洋舰专属节点/遭遇类型（地图结构不动）。

## 技术方案要点

- **纯数据版本**：船体/模块/弹药/无人机/NPC 全走既有管线（`ships.js`/`modules.js`/`ammo.js`/`drones.js`/`npcs.js` 填表 + `skills.js` 四个新 gate key）；新 skill key 零版本升级（`merge` 浅覆盖回填惯例）；引擎预期唯一改动是 FR-3 的带宽求和判定（`maxDronesInSpace`/`canLoadDrone` 签名微调，三个调用点同步）。
- 新 gate skills 自动进入 `skillUnlocks` 反向扫描（SkillsPanel 免费获得解锁提示）；`describeRequiredSkills` 惯例覆盖新门槛文案。
- 巡洋舰 DPS 预算沿用 v0.12 分账原则：Vexor 武器填装 + 满编面板 ≤ 同级最强纯炮平台（Thorax）×110%；突击护卫按"驱逐火力 90% + 护卫机动"双锚点校验。
- 中型弹药蓝图、中型无人机蓝图进 `BLUEPRINTS`（若 v0.16 已实施则带 `lootOnly` 分组，未实施则 T1 常驻组）——两版顺序无关，后者补联动。
- balance.md 新增"巡洋舰锚点"与"T2 船体锚点"两节 + TTK 参照带更新（巡洋舰 vs d13+ 基准 NPC 仍落 20–60s 带的 1.5 倍内）。
- 验证：Node 测试覆盖新船体装配矩阵（PG/CPU/槽位/技能门槛）、混编带宽矩阵（中 ×2 + 轻 ×4 等）、中型弹药修正矩阵与消耗锚点复算、DPS 预算表全船级复跑、突击护卫双抗与 stacking penalty 互不干扰。

## 验收标准

- [ ] `cruisers` 1 级 + 620,000 ISK 登舰 Thorax：16 槽装配、中型炮射程/伤害面板达标；无技能玩家可购不可登（boarding 门槛回归）。
- [ ] Vexor 混编：中型 ×2 + 轻型 ×4 在空成立（bw 50）、中型 ×5 被带宽拒绝；`drones` 技能等级仍是并行上限。
- [ ] 中型弹药：装填/消耗/换弹/结算与小型线行为一致；一场 d13 巡逻的弹药净消耗仍落赏金 5–15% 带。
- [ ] Enyo 对 d13 NPC：TTK 落带且被 Boss 级命中率显著低于巡洋舰（sig/速度差可测）——"小而精"路线成立。
- [ ] 五派系巡洋舰 NPC 在 d13+ 出现、可任编队主位、掉落表含中型件（FR-5 未滑落时）。
- [ ] 既有护卫/驱逐全部装配、战斗、经济行为零回归（老档载入零迁移）。

## 风险与开放问题

- **数值通胀对撞深度曲线**：巡洋舰 EHP ×3 遇上 `1 + 0.15×(depth−1)` 线性缩放，可能把 d10–15 打成无聊区（船太硬）——先按锚点落地，若实测如此，调 NPC 巡洋舰化的解锁密度（FR-5 权重）而不动缩放公式。
- **内容量是第三周期之最**（3 + 2 船体、10+ 模块、7 弹药、4 无人机、5 NPC、4 技能）：FR-5 明确可滑落；FR-2 的 Rail 线若工期紧可先出 Blaster 线（Thorax/Enyo 先行，Caracal/Harpy 随补丁）。
- **弹药 family 翻倍**（3 → 5）：FittingWindow 选择器与市场货架的信息密度再抬档，FR-6 复查必须含此项。
- `assault_frigates` 的解锁语境：EVE 里 AF 需要护卫 V，本作无 `frigates` 技能——开放问题：是直接 1 级门槛（训练成本 1,000 SP，偏便宜）还是要求 3 级（6,000 SP，用成本模拟资历）。倾向后者，实施计划定夺。
- 中型无人机的敌方转火权重是否与轻型同参（更大更慢 = 更容易被打），`DRONE_AGGRO_*` 三常量是否需要按体型分化——先同参观察，锚点复算后再议。
