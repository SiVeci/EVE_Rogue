# PRD v0.3 — 经济闭环：市场、首批模块变体、Kestrel 解锁

## 概述

在 StationHub 新增 MARKET 标签页：用 ISK 购买模块与船体。同步补充第一批"现有类型内"的模块变体（纯数据），解锁 Kestrel 作为第二艘可用船，并让船体损失真正计入死亡惩罚。

## 背景与动机

- **经济软锁**：`shipDestroyed()` 销毁全部已装配模块，但蓝图只能制造火箭发射器和护盾回充器。玩家带着爆能炮阵亡后将永久无法再获得爆能炮。市场覆盖全部 T1 模块可根治此问题。
- **ISK 没有出口**：目前 ISK 除制造两种模块外无处可花，奖励数值失去意义。
- BattleScene 已实现全部 6 种模块行为类型，同类型内加变体是零引擎成本的数据工作；**市场货架需要商品**，两者应同版本落地。
- Kestrel 数据已在 `ships.js` 完整定义，只缺购买/换船 UI；导弹+盾与爆能炮+甲是两套装配哲学，能把决策空间翻倍。

## 目标

- 玩家任何时刻都能用 ISK 重建一套完整装配（消灭软锁）。
- 死亡惩罚 = 船体 + 装配模块，但永不导致无法继续游玩。
- 玩家拥有两条可行的 build 路线（Incursus 爆能炮/甲修 vs Kestrel 导弹/盾回）。

## 功能需求

### 市场

- **FR-1 MARKET 标签页**：StationHub 第四个 tab。分类列出全部 T1 模块与船体，显示名称、类型、槽位、PG/CPU、关键属性、价格；点击购买，ISK 不足时禁用。
- **FR-2 出售**：机库中的模块可按买价 50% 卖出（防止误购锁死资金）。
- **FR-3 定价**：以收益锚定（深度 1 巡逻 = 15,000 ISK）。建议基准：T1 武器/EWAR 4,000–8,000；修理/推进 6,000–12,000；Incursus 60,000；Kestrel 75,000。数值允许开发中调整，原则是"一次阵亡 ≈ 3–5 场巡逻的净损失"。

### 船体系统

- **FR-4 船体资产**：store 新增 `ownedShips`（船体 id 列表，初始 `['incursus']`）。购买船体加入列表。
- **FR-5 换船**：FITTING 页可在拥有的船体间切换；切换时当前已装配模块全部自动卸回机库，再载入目标船体（空装配）。
- **FR-6 船体损失**：`shipDestroyed()` 改为同时从 `ownedShips` 移除当前船体。返回空间站后必须选择/购买船体才能出击。
- **FR-7 保底机制**：若玩家没有任何船体且 ISK < 最便宜船体价格，免费获得一艘 Incursus（EVE 新手船逻辑），保证永不卡死。

### 首批模块变体（纯数据，全部落在已有 6 类型内）

- **FR-8** 在 `modules.js` 新增约 7 件，形成"短程高伤 vs 远程低伤"和"标准 vs Meta"两条轴：

| 模块 | 类型/槽位 | 定位（建议数值可调） |
|------|-----------|----------------------|
| Light Ion Blaster I | hybrid_weapon / high | 电子炮上位：伤害 +25%，PG/CPU 更高 |
| 125mm Railgun I | hybrid_weapon / high | 远程炮：optimal ~9,000m，falloff 3,000m，tracking 低，DPS 约为爆能炮 60% |
| Light Missile Launcher I | missile_weapon / high | 远程导弹：range ~22,000m，DPS 约为火箭 65% |
| Medium Shield Booster I | shield_repair / mid | 超规格盾回：回量 ×2，PG/电容消耗大 |
| Small Armor Repairer I (meta) | armor_repair / low | 同回量，电容消耗 −25% |
| Fleeting Stasis Webifier | ewar / mid | 减速 55%→60%，CPU 更高 |
| 1MN Afterburner I (meta) | propulsion / mid | 速度倍率 2.15→2.35，电容 −20% |

- **FR-9 INDUSTRY 联动**：现有两张蓝图保留；蓝图制造价应低于市场直购价约 20–30%，维持工业页存在意义（蓝图扩充放 v0.6+）。

## 非目标

- MWD（需要信号半径惩罚模拟，且当前敌人用导弹、信号无意义 → 推迟到 v0.4 与炮台敌人同版本）。
- 被动模块（护盾扩展、装甲板等，需要属性聚合系统 → v0.5）。
- 市场价格波动、NPC 收购单。

## 技术方案要点

- `gameStore.js` 新增 `buyModule` / `sellModule` / `buyShip` / `switchShip` action；`shipDestroyed` 扩展船体销毁 + 保底。
- 新组件 `src/components/MarketWindow.jsx`，StationHub 加 tab；换船 UI 可并入 FittingWindow 顶部。
- 存档版本号 +1（新增 `ownedShips` 字段），写 v0.2 的 `migrate`。
- 模块变体仅改 `modules.js`；Railgun 的低 tracking 依赖 BattleScene 已有炮台命中公式，无引擎改动。

## 验收标准

- [ ] 全部 T1 模块与两艘船体可在市场购买/出售，ISK 正确扣减。
- [ ] 阵亡 → 船体消失 → 重买/保底 → 重新装配 → 可再出击，全流程无卡死。
- [ ] 身无分文 + 无船时自动获得免费 Incursus。
- [ ] Kestrel 装 4 发射器 + 盾回 + 加力可通关深度 1–3（手动验证 build 可行性）。
- [ ] 换船时装配模块全部回到机库，无凭空消失/复制。
- [ ] 旧版存档（v0.2 格式）加载后自动补上 `ownedShips: ['incursus']`。

## 风险与开放问题

- 定价与死亡惩罚强度需要实际游玩校准；先按 FR-3 原则落地，允许后续热调。
- 换船时若目标船槽位更少，多余模块直接回机库即可（本设计已覆盖）。
