# PRD v0.14 — Boss 节点与 run 终点结构

## 概述

给 run 一个"赢"的定义：里程碑深度（首个 d15，之后每 3 段一个）的段末层替换为 **Boss 节点**——巡洋舰级旗舰 NPC 带护卫编队、阶段机制与保证掉落；击败首个 Boss 触发"通关"结算画面，之后可继续下潜进入更深循环。这是 v0.9 PRD 明文留下的"段末守门人原型化"候选的兑现，也是无限爬深第一次有了节奏终点。

## 背景与动机

- v0.9 非目标原文："Boss 节点 / 段末守门人原型（候选池——段末先用 elite 顶位，原型化留给后续内容版本）"。段末 elite 是占位，现在补正主。
- 爬深无限且无胜利条件：长线动机全靠数值增长，roguelike 的"这局打穿了"体验缺失。首个 Boss = 首次通关里程碑，给新玩家一个 30–60 分钟可达的目标弧线。
- 依赖 v0.13 硬性成立：Boss + 护卫编队直接复用多敌引擎（Boss 是编队主位成员，护卫是席位成员），本版**不新增任何实体维度**——只新增数据形态（巡洋舰级 NPC、阶段块）与地图规则（里程碑段末替换）。
- Boss 的巡洋舰级数值顺路为 v0.17 玩家巡洋舰立第一批敌侧参照（大 sig / 慢速 / 高 HP 的船级形状先在 NPC 侧观察）。

## 功能需求

- **FR-1 Boss NPC 原型**：3 个 Boss（Guristas / Sansha / Serpentis 旗舰风味各一），巡洋舰级形状：sig 110–130m、速度 160–200 m/s、HP ≈ 同深 elite 护卫 NPC ×4–5、DPS ×1.5–2；保留派系弱抗签名（幅度收窄为 15–20 点——Boss 该更难被"答案"秒掉，但答案仍然成立）。锚点全套入 balance.md 新"Boss 带"一节。
- **FR-2 Boss 编队**：Boss 固定为编队主位 + 1–2 个同派系护卫成员（v0.13 席位系数照常适用于护卫，Boss 本体不吃系数）；护卫先死后 Boss 无行为变化（协同性仍由组合表达，不写脚本）。
- **FR-3 阶段机制**：NPC 数据新增可选 `phases: [{ hpPct, statMods, log }]`——Boss 结构层跌破阈值（50%）触发一次性属性切换（如狂暴：速度 ×1.3 / 射速 ×1.25，或换弹头伤害类型），战斗日志播报。只做属性/行为参数切换，不做召唤增援（与 v0.13 "所见即所遇" 冲突）。
- **FR-4 里程碑排布**：`generateRunMap` 在段末层 depth 命中里程碑（d15、d30、d45…）时生成单个 `boss` 节点替换 elite 层（该层恒 1 节点，全边汇入）；节点 intel 完整展示 Boss + 护卫构成与阶段提示（"结构受损时进入狂暴"一行风味文本）。非里程碑段末保持 elite 现状。
- **FR-5 保证掉落与奖励**：Boss 专属 `lootTable`：保证位（chance 1.0）一件当深品级上限模块（d15 = Meta 保证、d30+ = T2 保证）+ 常规位若干；赏金/SP ≈ 同深 elite ×3。击败即正常走 `advanceSegment`。
- **FR-6 通关结算与记录**：首次击败任意 Boss 弹出通关结算画面（本局深度、累计 ISK/SP、击杀数——继续下潜 / 回站两个出口）；store 新增 `bossKills: { [bossId]: count }` 与 `maxBossDepth`，站内可见（简单一行战绩，不做成就系统）。

## 非目标

- Boss 轮换池 / 每派系多 Boss 的内容量（3 个原型先行，扩充走后续内容补丁）。
- 召唤增援、位置机变（AOE、冲锋）等脚本化 Boss 机制——阶段属性切换是本版机制复杂度上限。
- 剧情文本、结局动画、成就系统。
- Boss 保险/特殊死亡惩罚变化（阵亡规则与普通节点完全一致）。
- 玩家巡洋舰（v0.17）。

## 技术方案要点

- 地图侧：`runmap.js` 新节点类型 `boss`，里程碑判定 `isBossDepth(depth)` 纯函数（`depth % 15 === 0`，常量可调）；App 选点分支把 `boss` 按战斗节点处理（复用 `pendingNodeId` 流程），胜利路径与段末 elite 相同（`advanceSegment` + 回站）——唯一新增是首杀时先插通关画面。
- Boss 数据仍是 `npcs.js` 普通条目 + `class: 'Cruiser'` 风味字段与 `phases` 块；`buildEncounter` 对 boss 节点跳过 elite ×1.5（Boss 自带体格，不再吃 elite 乘子，深度乘子照常）。
- 阶段切换在 BattleScene 敌成员帧循环内做一次性阈值检查（ref 记录已触发相位，镜像 `powered` 转换日志惯例）；statMods 直接改该成员的运行时 stats 副本，不回写数据对象。
- store：`bossKills`/`maxBossDepth` 增量 key，`partialize`/`merge` 缺省回填——**零版本升级**。
- 通关画面是 App 层 modal（复用 victory modal 骨架），不是第四视图。
- 验证：Node 测试覆盖里程碑排布（多种子断言 d15 段末恒 boss、d10/d20 段末恒 elite）、Boss 编队生成 shape、阶段触发一次性、保证掉落位、`bossKills` 持久化往返。

## 验收标准

- [ ] 起始深度 d11 的段：段末层为单个 boss 节点，intel 显示 Boss + 护卫与阶段提示；d6 起始段段末仍为 elite（回归）。
- [ ] Boss 战：护卫可先行击杀（v0.13 目标切换），Boss 50% 结构触发狂暴且日志播报；TTK 落在 90–180s 带（普通节点 20–60s 的 3 倍级）。
- [ ] 击败 d15 Boss：保证 Meta 掉落 + 赏金/SP ≈ elite ×3；首杀弹通关画面，`maxBossDepth` 入档；选择继续下潜则下段从 d16 生成。
- [ ] Boss 节点阵亡/撤退行为与普通节点规则完全一致（撤退封节点、段可能提前结束、深度不进）。
- [ ] 刷新页面丢段不丢 `bossKills`（存档往返测试）。

## 风险与开放问题

- **Boss 数值无先例**：巡洋舰级 HP/DPS 带是拍的，TTK 90–180s 目标带靠 Node 模拟先行校准，实机体感修正留给冒烟验证——调参杆固定为 HP 与 DPS 两个乘子，不动阶段机制。
- **d15 作为首个里程碑的节奏**（3 段 ≈ 12–15 场战斗）：太远则通关动机接不上新手期，太近则 Boss 沦为普通墙——若实测偏长，优先降为 d10（改常量），不缩段长。
- **撤退是否对 Boss 免费**：8 秒对齐对慢速 Boss 几乎无风险，"打不过就撤"会让 Boss 失去守门感——但撤退封节点 = 段提前结束、深度不进，本身已是重代价。先不加专项阀门，观察后再议（候选：Boss 节点对齐时间 ×1.5）。
- 保证掉落与 v0.16 蓝图掉落的关系（Boss 是否保证掉蓝图）留给 v0.16 排期时决定。
