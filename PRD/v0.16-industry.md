# PRD v0.16 — 蓝图掉落与工业深化

## 概述

INDUSTRY 从"弹药批产"扩展为真工业：**蓝图成为第四类掉落物**——Meta/T2 模块与无人机的蓝图低概率进 NPC 掉落表，捡到即永久解锁对应制造条目（阵亡不失）；制造沿用 75% 折价惯例，让高品级装备第一次有"可重复获取"的渠道，同时不动"Meta+ 只出自掉落"的精神（蓝图本身就是掉落）。

## 背景与动机

- v0.6 起 Meta/T2 只出自掉落：捡到一件 T2 很兴奋，但**丢了就只能再赌掉落**——阵亡损失全部已装模块的规则下，深层 build 的重建成本是纯运气，roguelike 的"损失→重建→再出发"循环在高品级段断裂。蓝图把"再赌一次"变成"攒钱重造"，损失依然痛（ISK 大额支出）但可规划。
- v0.11 已顺路把 INDUSTRY 从写死蓝图改为读 `src/data/blueprints.js` 静态数据（PRD 原文"为将来蓝图掉落铺路，本版不做掉落"）——数据层的门当时就留好了，本版走进去。
- ISK 常态出口目前只有弹药/注入剂（小额高频）；制造 Meta/T2 是中额低频出口，补上经济出口谱系的中段。
- 无人机蓝图在 v0.12 计划明文 out of scope（"drone blueprints are out of scope for v0.12 — market + loot only"），本版补齐——无人机是损耗品，恰恰是制造需求最刚性的品类。
- 依赖：v0.11（blueprints 数据文件 + `manufacture` 双路由）。与 v0.13–v0.15 无耦合，可插队实施。

## 功能需求

- **FR-1 蓝图物品与永久库**：蓝图是**知识而非物品**：store 新增 `blueprints: string[]`（已解锁蓝图 id 集合，永久持有、阵亡不失、`resetProgress` 才清空）。`BLUEPRINTS` 数据扩展为三组：T1 组（现状 2 模块 + 11 弹药 + 新增 4 种 T1 无人机蓝图，无需解锁、常驻可造），Meta 组与 T2 组（每条目带 `lootOnly: true`，须掉落解锁后可见可造）。
- **FR-2 蓝图掉落**：`rollLoot` 第四种条目 shape `{ blueprintId, chance }`：NPC 掉落表按派系分布 Meta/T2 蓝图条目（掉率 ≈ 对应成品模块条目的 1/3——成品仍是主要兴奋源，蓝图是长线保底）；蓝图掉落受成品同款 `TIER_UNLOCKS` 深度门控。已解锁的蓝图重复掉落时自动折为 ISK（面额 = 该蓝图制造成本的 25%，入 `notice`/victory modal 文案）。
- **FR-3 制造定价**：Meta/T2 制造成本 = 成品 `price` × 75%（沿用 v0.11 弹药蓝图的折价惯例）；无人机蓝图批量 1 架/次（T1 6,000 / Meta 9,000 / T2 21,000 ISK，= 各自市价 75%）。产出路由沿用 `manufacture` 现状三向判定：`MODULES` → inventory、`AMMO` → cargo、`DRONES` → droneHangar（第三路由是本版在 `manufacture` 里唯一的代码增量）。
- **FR-4 INDUSTRY UI 分组**：页签内按 弹药 / 模块 / 无人机 分组；未解锁的 Meta/T2 蓝图显示为占位行（名称 + "蓝图未获取"，不显示掉落来源——保留探索性），已解锁条目显示成本与产出。
- **FR-5 Boss 蓝图位（若 v0.14 已实施）**：Boss 保证掉落位增加蓝图备选（保证位 roll 成品或蓝图二选一）——两版排期顺序无关，后实施的一版补此联动。

## 非目标

- 材料/矿石系统（制造成本纯 ISK——引入第三种资源轴留给远期，先验证"蓝图解锁 + ISK 重造"的循环成不成立）。
- 制造时间/队列（即时交付，无时间系统）。
- 蓝图拷贝/次数限制（全部 BPO 语义：解锁即无限次）。
- 船体蓝图（船体损失曲线已有保险平滑，双保险会把阵亡痛感打穿地板）。
- 蓝图交易/出售（知识不可出售；重复掉落自动折 ISK 已覆盖多余价值）。
- 装备词缀/Abyssal roll（远期候选不变）。

## 技术方案要点

- store：`blueprints` 增量 key（id 数组，`partialize` 直存、`merge` 对未知 id 静默丢弃 + 缺省 `[]` 回填）——**零版本升级**。`manufacture` 增加 `DRONES` 路由与"蓝图已解锁"校验（T1 组免校验）。
- `rollLoot`（`src/lib/loot.js`）第四 shape 分支 + 重复蓝图折 ISK 的判定（需要读 store 已解锁集合——保持纯函数：由调用方把 `ownedBlueprints` 作为入参传入，测试惯例不破）。`addLoot` 第四路由：`{ blueprintId }` → `blueprints` 集合（幂等去重）。
- 三处掉落展示（BattleScene victory modal、MapScreen `NoticeOverlay`、App 残骸和平路径）按 shape 分支各加一行渲染——v0.11/v0.12 两轮同型改造的第三次重复，模式成熟。
- `TIER_COLORS`（`src/lib/tiers.js`）复用为蓝图品级着色；蓝图条目命名惯例 `{moduleId}_bp`，展示名 "XXX Blueprint"。
- balance.md 新增"工业"一节：75% 折价惯例成文、蓝图掉率 = 成品 1/3 的 EV 校准（模块 EV 带 0.35–0.45 不含蓝图，蓝图是补充位——同弹药束/无人机束惯例）、重复折价 25%。
- 验证：Node 测试覆盖第四 shape 路由/幂等解锁/重复折 ISK、`manufacture` 三向路由与未解锁拒造、存档往返（蓝图集合 + 未知 id 丢弃）、掉落表 EV 复跑（模块带不漂移）。

## 验收标准

- [ ] 深度 4+ 击杀掉落 Meta 蓝图：victory modal 显示蓝图行（品级着色），INDUSTRY 对应条目从占位变为可造。
- [ ] 制造 Meta 模块：扣 ISK = 成品价 75%，产出进 inventory；制造 Hornet 进 droneHangar；弹药批产行为与 v0.11 完全一致（回归）。
- [ ] 阵亡：`blueprints` 集合完整保留（与 cargo/droneHangar 同规则）；`resetProgress` 后清空、T1 组常驻可造。
- [ ] 重复掉落已解锁蓝图：自动折 ISK 且文案可见；同一蓝图永不重复占用库容。
- [ ] 未解锁 T2 蓝图的条目不可制造（UI 占位 + `manufacture` 拒造双保险）。
- [ ] 老档载入：`blueprints` 缺省回填空集合，其余行为零回归（无版本升级验证）。

## 风险与开放问题

- **制造渠道稀释掉落兴奋感**：若蓝图解锁太快，"捡到 T2"退化为"反正能造"——1/3 掉率与 75% 定价是两道闸；若实测解锁曲线过快，先降掉率（单杆），不动定价惯例。
- **ISK 通缩压力**：中额出口加入后，中期结余可能从堆积转为紧张——观察赏金/支出比，必要时调市场售回率（50%）而非赏金基数。
- 占位行"不显示掉落来源"是探索性与可规划性的取舍——若玩家反馈"不知道去哪刷"挫败感强，回退为显示派系提示（一行字的改动量）。
- `rollLoot` 入参增加 `ownedBlueprints` 触碰三个调用点（BattleScene victory、runmap 残骸预掷、App 伏击路径）——残骸预掷发生在拥有状态可能变化之前（段中捡到蓝图），接受"预掷时点的快照"语义（与 encounter 预掷同理），PRD 明示不算 bug。
